# V0.4.5 — 本地学习报告 (Implementation Plan)

> Status: in progress  
> Branch: `feat/v0.4-deep-learning-and-spelling`  
> Author: assistant (autonomous)

## Goal

Give the kid (and parent) a single read-only screen that summarizes
**how learning is going so far**: overall accuracy, mastered word
count, today's review completion, and the weakest two or three
categories. The page is purely retrospective — no editing, no cloud
sync, no streaks/heatmap chart.

## Out of scope

- 7-day / 30-day trend graphs (would require time-series persistence
  V0.4.x doesn't have today).
- Per-question breakdown by `QuestionKind` — the kid never sees the
  internal Choice/FillLetter/Spell distinction directly.
- Cloud sync of the report.
- Editable goals / target accuracy.

## Decisions

1. **Entry point.** `TodayPlanPage` top bar gains a `📊` button
   labelled `TodayPlanReportButton`. Pushes `pages/LearningReportPage`.
   We resist a 6th HomePage toolbar icon — adding one would push the
   row over the comfortable 360 dp width budget; stacking the report
   under the plan page also reads as "today's plan ↔ how am I doing"
   which is the natural user-mental-model pairing.
2. **Stats computation.** A new `LearningReportBuilder` is a pure
   function: `(repo, recorder, nowMs) -> LearningReport`. It iterates
   the repo to discover all categories (so empty categories show
   zeros instead of being silently skipped), then folds in
   `WordStat` entries from the recorder. Mastery / familiarity counts
   come from `MemoryScheduler.classify` so the report always agrees
   with the rest of the app.
3. **Today's review completion.** A "review-eligible" word is one
   whose stored `memoryState` is in {`learning`, `familiar`,
   `mastered`} AND was last answered before today (i.e. there is
   actually something to review today). "Done today" is the V0.4.4
   signal — `lastAnsweredMs >= localStartOfDay(now) &&
   consecutiveCorrect > 0`. The completion ratio is
   `done / max(due, done)` so the bar never overflows when the kid
   reviews words that weren't due.
4. **Weak categories.** Sorted ascending by accuracy
   (`correct / seen`). Categories with `seen == 0` are excluded
   (no signal). The top-2 weakest are surfaced; if fewer than two
   categories have data, we render whatever's available.
5. **Empty / fresh-install state.** When the recorder has no stats,
   the report shows "今天还没开始学习呢" + zero counts. No crashes.
