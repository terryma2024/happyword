# V0.7.0 Monorepo Native Clients Restructure Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move the existing HarmonyOS DevEco project into `harmonyos/`, prepare root-level native client slots for `ios/` and `android/`, and keep `server/` plus shared contracts as first-class monorepo modules.

**Architecture:** The repository root becomes a monorepo workspace. `harmonyos/` becomes the complete DevEco Studio project root, while root-level docs, scripts, tools, and CI coordinate modules. iOS and Android remain native-client placeholders in this migration; no product functionality is added.

**Tech Stack:** HarmonyOS NEXT / ArkTS / ArkUI / DevEco Studio / Hvigor / OhPM, Python FastAPI server with uv, Bash tooling, Markdown documentation.

---

## File Structure

### Create

- `harmonyos/`: new DevEco project root containing the moved HarmonyOS files.
- `ios/README.md`: native iOS placeholder and scope guard.
- `android/README.md`: native Android placeholder and scope guard.
- `shared/README.md`: shared contracts / fixtures scope guard.
- `shared/contracts/.gitkeep`: keeps contracts directory tracked until schema extraction starts.
- `shared/fixtures/.gitkeep`: keeps fixtures directory tracked until golden fixture extraction starts.

### Move

- `AppScope/` -> `harmonyos/AppScope/`
- `entry/` -> `harmonyos/entry/`
- `hvigor/` -> `harmonyos/hvigor/`
- `hvigorfile.ts` -> `harmonyos/hvigorfile.ts`
- `build-profile.json5` -> `harmonyos/build-profile.json5`
- `oh-package.json5` -> `harmonyos/oh-package.json5`
- `oh-package-lock.json5` -> `harmonyos/oh-package-lock.json5`
- `code-linter.json5` -> `harmonyos/code-linter.json5`

### Modify

- `AGENTS.md`: monorepo instructions and `harmonyos/` working directory.
- `CLAUDE.md`: mirror `AGENTS.md`.
- `README.md`: monorepo overview and per-module commands.
- `.cursor/dev-commands.md`: Harmony command source of truth with `harmonyos/` as project root.
- `scripts/run_ui_tests.sh`: introduce `HARMONY_ROOT`, run Hvigor from `harmonyos/`, find HAPs under `harmonyos/entry/build`.
- `tools/recraft/start-icon-to-launcher.sh`: write launcher PNGs under `harmonyos/AppScope` and `harmonyos/entry`.
- `tools/recraft/generate-bosses.sh`: write character SVGs under `harmonyos/entry`.
- `tools/generate_scan_binding_qr_fixture.py`: write QR fixture under `harmonyos/entry`.
- `docs/WordMagicGame_overall_spec.md`: update command-critical project structure and test path references.
- `docs/WordMagicGame_roadmap.md`: update command-critical path examples introduced before V0.7.0.
- `docs/server/operations.md` and `docs/server/parent_web_operations.md`: update any command-critical Harmony paths if present.

### Do Not Move

- `server/`
- `assets/`
- `generated/`
- `plugins/`
- `scripts/`
- `tools/`
- `docs/`
- `.idea/`, `.hvigor/`, `oh_modules/`, `build-tmp/`, `local.properties`

`local.properties` is a local-only handoff file. Copy it manually to `harmonyos/local.properties` during execution, but do not stage it.
