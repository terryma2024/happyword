# V0.5.2 ‚Üí V0.5.7 Implementation Plan (server-side)

> **Mode:** Inline execution by author. Per-sub-version commit. TDD throughout.
> **Branch:** `feat/v0.5.2-thru-v0.5.7` (off `feat/v0.5-remaining`).
> **Spec:** [`2026-04-30-v0.5-content-backend-design.md`](../specs/2026-04-30-v0.5-content-backend-design.md)
> **Roadmap:** [`WordMagicGame_roadmap.md` ¬ß12](../../WordMagicGame_roadmap.md)

**Goal:** Land V0.5.2 through V0.5.7 on the FastAPI server: word CRUD, versioned pack publish/rollback, LLM word-level drafts, lesson photo import, Vercel Blob assets, and ops wrap-up. Commit per sub-version. Client-side ArkTS work (HarmonyOS) is intentionally deferred ‚Äî the spec marks V0.5.2 as zero-client-change and the rest as additive client work that can land in a follow-up.

**Architecture:** Stay inside the existing `server/app/` layout (config / deps / models / schemas / services / routers). One Beanie `Document` per new collection, one router per admin sub-area, one service module per LLM/Blob boundary. All schemas grow forward-only; older clients ignore unknown fields. Vercel Blob is talked to via `httpx` directly (no extra SDK), per spec ¬ß4.1.

**Tech stack:** Python 3.12 ¬∑ FastAPI ¬∑ Beanie + motor ¬∑ Pydantic v2 ¬∑ OpenAI 2.x (vision + structured outputs) ¬∑ Vercel Blob HTTP API ¬∑ pytest + pytest-asyncio + mongomock-motor ¬∑ uv.

**Project rule (non-negotiable):** every commit that touches `server/` must run `uv run pytest` with **0 errors and 0 warnings**. `pyproject.toml` has `filterwarnings = ["error", ...]` so any new warning fails the suite. New narrow ignores must pin both message + module and explain why upstream cannot be fixed.

## TDD Discipline (applies to every sub-version)

For each new endpoint / behaviour:

1. **RED** ‚Äî write the smallest pytest case that asserts the desired behaviour. Run it ‚Üí expect failure with a clear "feature missing" error (e.g. `404 not found`, `ImportError`, `AttributeError`). If the test passes immediately, the test is wrong; rewrite.
2. **GREEN** ‚Äî write the minimum production code that satisfies the test (and only that test).
3. Re-run only the new test, then the full suite, to confirm 0 regressions and 0 warnings.
4. Move to the next behaviour.

Mocking conventions:
- `OPENAI_API_KEY=""` in autouse `_env` fixture ‚Üí tests that exercise LLM paths monkeypatch the *imported* function in the router module (e.g. `monkeypatch.setattr(admin_drafts, "extract_word_distractors", fake)`), never the source module. Mirrors `test_admin_llm_endpoint.py`.
- Vercel Blob ‚Üí tests monkeypatch `app.services.blob_service.upload_blob` to return a deterministic `https://stub.blob/...` URL.
- Mongo ‚Üí `mongomock-motor` via the shared `db` / `client` fixtures in `tests/conftest.py`.

Linting gate after every commit:

```bash
uv run pytest         # 0 failed, 0 warnings, nothing skipped except live_openai
uv run ruff check .
uv run ruff format --check .
uv run mypy app
```

Commit message convention: `feat(server): V0.5.x <one-line summary>`. Body bullets list new endpoints, new collections, schema bumps, and notable test counts.

**Endpoints (all `/api/v1/admin/words`, admin JWT required):**

| Method | Path | Behaviour |
|---|---|---|
| GET | `/admin/words` | Paginated list. `?category=&difficulty=&q=&page=1&size=50`. Excludes soft-deleted by default; `?include_deleted=true` to opt in. |
| GET | `/admin/words/{id}` | Single Word. 404 if missing or soft-deleted. |

| PUT | `/admin/words/{id}` | Partial merge of allowed fields. 200 + `WordOut`. 404. |

**Behaviour contracts (test list, ‚â•10):**

2. parent role ‚Üí 403 on every endpoint
3. POST creates a word; GET-by-id returns it
4. POST with duplicate id ‚Üí 409 `DUPLICATE_ID`
5. POST with bad payload (missing field, difficulty out of 1-5) ‚Üí 422
6. PUT partial only updates given fields; `updated_at` advances
7. PUT 404 when id missing
8. DELETE marks `deleted_at`; subsequent GET returns 404; `?include_deleted=true` returns it
9. DELETE on already-deleted ‚Üí 404
10. soft-deleted word is excluded from `/api/v1/packs/latest.json`
11. seed script: dry run on empty DB inserts 50 words, second run is a no-op (count stays 50)
12. seed script: pre-existing word with same id is **not** overwritten (admin edits win)

**Acceptance:**
- `uv run pytest` ‚Üí all green, 0 warnings.
- `uv run python scripts/seed_from_rawfile.py` (against a real local Mongo or `--dry-run`) reports 50 inserted on a clean DB.

## V0.5.3 ‚Äî Versioned pack publish + rollback

**Goal:** Snapshot publishing model. Each `POST /admin/packs/publish` writes a `WordPack` doc and bumps the `PackPointer`. `/api/v1/packs/latest.json` reads the pointer and serves with `ETag` + `If-None-Match ‚Üí 304`.

**New files:**
- `app/models/word_pack.py` ‚Äî `WordPack(version: int unique, words: list[dict], schema_version: int, published_at, published_by, notes?)`.
- `app/models/pack_pointer.py` ‚Äî `PackPointer(singleton_key: Literal["main"]="main" unique, current_version: int, previous_version: int | None)`.
- `app/schemas/admin_pack.py` ‚Äî `PublishIn(notes?)`, `PublishOut`, `RollbackOut`, `PackListItem`, `PackListOut`.
- `app/services/pack_service.py` ‚Äî `publish_pack(by_username, notes) -> WordPack`, `rollback_pack() -> PackPointer`, `serialize_words_for_pack(words) -> list[dict]`, `get_current_pack() -> WordPack | None`.
- `app/routers/admin_packs.py` ‚Äî `/admin/packs/{publish,rollback,current}` + list + get-by-version.
- `tests/test_packs_publish.py` ‚Äî publish v1, edit a word, publish v2, rollback to v1.
- `tests/test_packs_etag.py` ‚Äî `If-None-Match: "1"` returns 304; mismatched ETag returns 200 + new ETag.
- `server/scripts/publish_pack.py` ‚Äî convenience CLI used by ops.

**Modified files:**
- `app/main.py` ‚Äî register `admin_packs.router`; `init_beanie` document_models += `[WordPack, PackPointer]`.
- `app/routers/public_packs.py` ‚Äî `latest_pack` reads pointer ‚Üí snapshot when present, falls back to live `Word.find_all()` when no pack has ever been published (dev-friendly bootstrap). Emits `ETag: "{version}"`. Honors `If-None-Match`.
- `app/schemas/pack.py` ‚Äî keep schema v1 wire shape; pack JSON serialization moves into `pack_service`.
- `tests/conftest.py` ‚Äî add `WordPack`, `PackPointer` to `init_beanie` document_models.

**Behaviour contracts (‚â•8):**
1. publish on empty Word collection ‚Üí 409 `EMPTY_PACK`
2. publish with 1 word ‚Üí version=1, pointer.current=1, previous=None
3. edit word + publish ‚Üí version=2, pointer.current=2, previous=1
4. publish with `notes` stores notes
5. rollback ‚Üí pointer flips: current=1, previous=2; `latest.json` returns v1 words
6. rollback when previous=None ‚Üí 409 `NO_PREVIOUS_VERSION`
7. `/packs/latest.json` includes `ETag` header equal to `"{version}"`
8. `If-None-Match: "{current_version}"` ‚Üí 304, no body
9. soft-deleted words (V0.5.2) are excluded from snapshots at publish time
10. non-admin ‚Üí 401/403 on publish/rollback

**Acceptance:**
- `uv run pytest` green, 0 warnings.
- Local manual: seed ‚Üí publish ‚Üí curl `/packs/latest.json` returns v1 + `ETag: "1"`. Edit a word ‚Üí publish ‚Üí v2. Rollback ‚Üí v1.

**Commit:** `feat(server): V0.5.3 versioned pack publish + rollback + ETag/304`

**Endpoints (all admin):**
| Method | Path | Behaviour |
|---|---|---|

**Behaviour contracts (‚â•10):**

8. approve same draft twice ‚Üí 409 `ALREADY_REVIEWED`
9. reject ‚Üí word unchanged; draft.status="rejected"
10. PATCH while pending updates content; approve uses edited content
11. PATCH after approve/reject ‚Üí 409
12. non-admin 401/403

**Acceptance:**
- `uv run pytest` green, 0 warnings.
- Pack JSON for an upgraded word includes `distractors` + `example.{en,zh}` and `schema_version: 2`. Words without these fields are unaffected.
