## Goal

Deliver the parent-side vocabulary loop and child merged vocabulary fetch:

- Create / list / archive family pack definitions (existing + unchanged contracts).
- Draft edits: single-word API (existing), **batch upsert**, **image → draft import**.
- Publish / rollback (existing JSON routes); HTML publishes via thin form POST.
- Child device: **global + family merged** JSON at **`/api/v1/child/packs/latest.json`** with tenant isolation and optional **`X-Family-Id`** guard.

**Out of scope:** V0.8.2 **`/admin/`** console (separate plan).

## HTTP contracts (summary)

### `POST .../draft/words:batch-upsert`

- Body: `{ "rows": [ { "word_id", ...FamilyPackDraftWordIn fields }, ... ] }`
- **207** when any row errors; **200** when all rows succeed.
- Body always includes `accepted_count`, `error_count`, `draft`, `errors[]`.

### `POST .../import-image`

- Multipart **`image`**; MIME **`image/jpeg`**, **`image/png`**, **`image/webp`**; max **8 MiB** (handler constant).
- **201** + `FamilyPackImportImageOut` on success.

### `GET`/`HEAD` `/api/v1/child/packs/latest.json`

- **`ChildPacksMergedOut`**: includes **`global_version`**, **`family_versions`**, flat **`words`**, **`packs`** slices.
- **`403`** **`TENANT_MISMATCH`** if **`X-Family-Id`** present and ≠ binding’s **`family_id`**.
- **`204`** if merged **`words`** empty; otherwise **`ETag`**, **`Cache-Control: private, no-cache`**, **`304`** when **`If-None-Match`** matches.

### `GET`/`HEAD` `/api/v1/child/family-packs/latest.json`

- Unchanged semantic: **family-published packs only** (`FamilyPacksMergedOut`). Not a substitute for the merged global+family endpoint.
