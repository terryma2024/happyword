# V0.4.8 Polish Pass Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans
> to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Six small visual / audio polish fixes that close out V0.4 — a
review-icon rendering bug, a friendlier wishlist toolbar icon, a sharper
launcher icon, a fight-pose magician swap, a word-on-projectile readability
upgrade, and a player-hurt SFX.

**Architecture:**

- All six changes are confined to existing UI surfaces; no new pages, no
  new persistent state, no schema migrations.
- We keep V0.3.10's "rawfile SVG icons + Recraft V4 vector toolchain" as
  the asset pipeline. New artwork is generated through
  `tools/recraft/generate-v4-vector.mjs` so the prompts stay reviewable.
- `MagicProjectile` becomes word-aware via a new `@Prop label` so
  BattlePage can pipe in the active answer at pulse time without
  mutating the projectile's animation lifecycle.
- The fight-pose swap is driven by a new `@State playerFighting:
  boolean` in BattlePage. It flips on at pulse time and back off after
  `SPAWN_MS + TRAVEL_MS + IMPACT_MS` (560 ms) so the magician returns
  to idle exactly when the impact frame ends.

**Tech Stack:** ArkTS / ArkUI (HarmonyOS NEXT), Recraft V4 Vector
(SVG generation), librsvg (rsvg-convert for PNG rasterization),
Hypium (unit + UI tests).

## Decisions (record once, reuse below)

1. **Review-icon bug → `.syncLoad(true)`.** The review SVG is 58 KB
   with 119+ paths. ArkUI's default `Image` is async; on second mount
   (back from BattlePage / WishlistPage / etc.) the rasteriser doesn't
   block the layout pass and the user catches the `Image` with no
   bitmap rendered for one to two frames — perceived as "the icon
   went back to a placeholder/emoji." `CharacterCard` already uses
   `.syncLoad(true)` for exactly this reason; we apply the same to
   all five toolbar `Image` children (review, codex, wand→gift, gear,
   scroll on WishlistPage). HomePlanButton stays a `Text('📋')`
   emoji — that's outside the user-reported scope and changing it
   would couple V0.4.8 to a Recraft regen for clipboard art.
2. **Wishlist icon = `wishlist.svg` (gift box).** We rename the
   button's image from `wand.svg` to `wishlist.svg` (a new file, not
   an overwrite, so we can revert quickly) and keep `wand.svg`
   committed for now. The Recraft prompt sits at
   `tools/recraft/prompts/wishlist-gift.txt`. The id
   `HomeWishlistButton` stays unchanged so existing UI tests keep
   working.
3. **Launcher PNGs → derived from `rawfile/icons/start_icon.svg`.**
   The current `tools/recraft/icons-to-launcher.sh` expects three
   files in `generated/recraft/icons/`. We extend it to accept a
   single source mode (`--from-rawfile`) that rasterises
   `entry/src/main/resources/rawfile/icons/start_icon.svg` to
   1024×1024 (foreground), pads it on a soft purple tint to 1024×1024
   (background), and 216×216 (startIcon). The simplest split: the
   foreground PNG is the SVG rasterised on a transparent background;
   the background PNG is a solid `#1D3557` 1024×1024; the startIcon
   PNG is the SVG rasterised at 216×216. Manifest paths
   (`layered_image.json` / `module.json5` / `app.json5`) stay
   unchanged.
4. **Fight-pose magician swap is BattlePage-local.** We don't change
   `PLAYER_CHARACTER_ASSET` in `CharacterAssets.ets` because the
   value is also read by the home / result pages where idle pose is
   correct. Instead BattlePage computes `currentPlayerAsset()` and
   passes that to `CharacterCard.assetPath`. ArkUI re-reads the prop
   when `playerFighting` flips, and `CharacterCard.Image.syncLoad`
   keeps the swap visually crisp.
5. **Projectile shape = `Ellipse` with the answer word centred.**
   We swap the `Circle` core for `Ellipse({ width, height })` (60×34
   at intensity=1, 84×48 at intensity=2). The outer glow stays a
   `Circle` because rotational symmetry reads better on a
   non-directional halo. A `Text(label)` overlay sits on top of the
   ellipse, fontSize 14 (combo: 18), white with a thin black
   `decoration({ type: TextDecorationType.None })` and `fontWeight:
   FontWeight.Bold`. Empty label hides the text.
6. **Player-hurt SFX = downloaded from Pixabay (CC0).** Pixabay's
   "sound effects" collection ships under the Pixabay Content
   License (free for commercial use, no attribution required). We
   download a short "ouch / grunt" sample, normalise to OGG via
   `ffmpeg` (already on macOS via Homebrew), commit as
   `entry/src/main/resources/rawfile/sound/player_hurt.ogg`. If
   ffmpeg is not on the developer's PATH the script falls back to a
   plain copy of the source MP3 (most AVPlayer builds accept MP3
   too, but OGG is preferred for parity with the existing samples).

- [ ] **Step 1: Create a sibling helper script `tools/recraft/start-icon-to-launcher.sh`.**

```bash
#!/usr/bin/env bash
# Rasterise the in-repo start_icon.svg into the layered-image PNGs.
#
# Inputs:
#   entry/src/main/resources/rawfile/icons/start_icon.svg
#
# Outputs (overwritten):
#   entry/src/main/resources/base/media/foreground.png  1024x1024 (transparent bg)
#   entry/src/main/resources/base/media/background.png  1024x1024 (#1D3557 solid)
#   entry/src/main/resources/base/media/startIcon.png    216x216
set -euo pipefail
cd "$(dirname "$0")/../.."

SRC_SVG="entry/src/main/resources/rawfile/icons/start_icon.svg"
DST="entry/src/main/resources/base/media"
TMP=$(mktemp -d)
trap "rm -rf $TMP" EXIT

if ! command -v rsvg-convert >/dev/null 2>&1; then
  echo "rsvg-convert not on PATH (brew install librsvg)." >&2
  exit 1
fi
if ! command -v magick >/dev/null 2>&1 && ! command -v convert >/dev/null 2>&1; then
  echo "ImageMagick (magick / convert) required for the solid background." >&2
  exit 1
fi
IM=$(command -v magick || command -v convert)

rsvg-convert -w 1024 -h 1024 -o "$DST/foreground.png" "$SRC_SVG"
"$IM" -size 1024x1024 xc:'#1D3557' "$DST/background.png"
rsvg-convert -w 216  -h 216  -o "$DST/startIcon.png"  "$SRC_SVG"

ls -la "$DST/foreground.png" "$DST/background.png" "$DST/startIcon.png"
```

```bash

```
