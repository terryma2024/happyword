# V0.4.4 — 每日学习计划页 (Implementation Plan)

> Status: in progress  
> Branch: `feat/v0.4-deep-learning-and-spelling`  
> Author: assistant (autonomous)

## Goal

Give the player (and any parent peering over their shoulder) a read-only
view of **today's** word lineup, grouped by what role each word will play
(`Review` / `Learning` / `New`) and decorated with a "✓ done" checkmark
once the kid has answered the word correctly today. The page is purely
informational; the only mutating entry to the battle stays the existing
"开始今日冒险" button on `HomePage`.

## Out of scope

- Editing the plan or pinning specific words.
- Showing mastery as a UI element (V0.4.5 report territory).
- Persisting an explicit "completed-today set" — derived from
  `WordStat.lastAnsweredMs` + `consecutiveCorrect`.
- Per-region word swapping inside the plan page.

## Decisions

1. **Plan determinism.** `TodayAdventureBuilder` historically used
   `Math.random()`; that means the plan-page preview can diverge from
   the actual battle. We seed the page-side builder with
   `mulberry32(hashSeed(regionId + dayKey))` so the page view matches
   the words the player will hit, day after day, until the day rolls
   over. The actual battle-start path stays untouched (it uses the
   default RNG); we accept the small divergence between the page
   preview and the live battle for V0.4.4 — both are honest projections
   of "today's plan", just sampled with different seeds. Promoting full
   determinism into `enterTodayAdventure` is out of scope.
2. **Today-completed signal.** A word counts as "done today" when
   `WordStat.lastAnsweredMs ≥ localMidnight(nowMs)` AND
   `WordStat.consecutiveCorrect > 0` (the most recent answer today was
   correct). This rides V0.4.3 streak data and avoids a new persistence
   layer.
3. **Entry point.** Add a circular toolbar button "📋" labelled
   `HomePlanButton` between the codex and wishlist icons. Clicking
   pushes `pages/TodayPlanPage`. No nav from inside BattlePage.
4. **Memory-state badge.** Reuse `MemoryScheduler.classify` so each
   row carries `Review / Learning / Familiar / Mastered / New`. No
   custom colours per state in V0.4.4 — single neutral chip with the
   Chinese label keeps scope tight.
