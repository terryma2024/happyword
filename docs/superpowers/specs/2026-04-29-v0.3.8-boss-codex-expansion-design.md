## 4. 战斗集成

### 4.1 TodayAdventureBuilder 轮换 seed

```typescript
function pickBossCatalogIndexForDay(region: AdventureRegion, dayKey: string): number {
  if (region.bossCandidates.length === 0) {
    return region.monsterPlan.slots[region.monsterPlan.slots.length - 1].catalogIndex;
  }
  const seedStr: string = `${region.id}:${dayKey}`;
  let hash: number = 5381;
  for (let i = 0; i < seedStr.length; i++) {
    hash = ((hash << 5) - hash) + seedStr.charCodeAt(i);
    hash = hash | 0;
  }
  const positive: number = Math.abs(hash);
  const idx: number = positive % region.bossCandidates.length;
  return region.bossCandidates[idx];
}
```

`build()` 中：

```typescript
plan.monsterSlots = region.monsterPlan.slots.slice();   // existing
const dayKey: string = localDayKey(nowMs);
const bossIdx: number = pickBossCatalogIndexForDay(region, dayKey);
const lastIdx: number = plan.monsterSlots.length - 1;
const oldSlot: MonsterSlot = plan.monsterSlots[lastIdx];
const newSlot: MonsterSlot = new MonsterSlot();
newSlot.kind = oldSlot.kind;          // remain MonsterKind.Boss
newSlot.catalogIndex = bossIdx;
plan.monsterSlots[lastIdx] = newSlot;
```

性质：
- 同一天同 region → 同一只 boss（`hash(regionId + dayKey)` 是确定函数）
- 不同 region 当天可见不同 boss（regionId 加进 seed 字符串）
- 不同天 → 哈希值变化 → 候选轮换；轮换周期是 `lcm(候选数, 实际 hash 分布)`，3 候选时通常 3 天一轮、2 候选时 2 天一轮
- 不动 `this.rng` —— word-pick 行为不变

| 风险 | 影响 | 缓解 |
|---|---|---|
| Witch 紫色违反 catalog hue 守卫 | MonsterCatalog 单测红 | spec §3.1 选 `#2B3470` 避开禁区；实现时跑测试再次确认 |

## 8. 非目标（V0.3.8 明确不做）

- 不增加新 region（4 个候选 boss 完全在战斗里"安家"是 v0.4 的事）。
- 不改 `MonsterPlan.slots.length`、不引入 Boss Rush 模式、不改 5-slot battle 结构。

- 不接 TTS 朗读 boss 名 / kindLabel。
- 不接 swipe / 长按 / 缩放手势（codex 仍只用左右箭头）。
- 不持久化 `currentCodexIndex`（每次进入仍从 Slime 开始）。
