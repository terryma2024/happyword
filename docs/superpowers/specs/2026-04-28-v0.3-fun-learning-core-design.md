# WordMagicGame V0.3 — 趣味学习内核版

- **Date:** 2026-04-28
- **Roadmap:** [WordMagicGame_roadmap.md §4](../../WordMagicGame_roadmap.md)
- **Status:** Design-for-implementation, single spec, phased plan (A → B → C)
- **Out-of-scope deferred to V0.3.5:** 学校城堡 / 家庭小屋两个区域、补字母题中级难度、今日冒险区域切换 UI

## 1. 范围与架构概览

### 1.1 V0.3 一句话

> 在水果森林这一个主题区域里，让孩子每天玩一局"今日冒险"：怪物种类决定题型，遗忘曲线决定哪些词被抽到，结算页给出魔法币，魔法币能在本地愿望单里换 3 件实物或时间奖励，长按 3 秒由家长确认。

### 1.2 实施阶段

V0.3 是单 spec、单实施计划、3 个阶段的代码落地。每个阶段都能独立合并并通过 CI（hvigor build + codelinter + hosTest）：

```text
Phase A — 题型与怪物 (基础设施)
  · 题型抽象 (Question.kind = 'choice' | 'fill-letter')
  · 补字母题生成器 FillLetterGenerator (初级:缺 1 字母 / 3 选 1)
  · 怪物类型枚举 MonsterKind + MonsterEntry.kind
  · BattleEngine 引入 IQuestionSource 接口 + submitAnswer 兼容 fill-letter 判定
  · QuestionGenerator 新增 nextQuestionForWord 辅助 (供 PlanQuestionSource 复用)
  · BattlePage 增加 LetterTemplate 渲染分支

Phase B — 今日冒险与记忆状态 (核心玩法)
  · WordStat 扩 lastCorrectMs / nextReviewMs / memoryState
  · MemoryScheduler 状态机 (新→学习→复习→熟悉→掌握)
  · LearningSnapshot v1 → v2 + 兼容读取
  · AdventureRegion 数据模型 + AdventureCatalog (内置 fruit-forest)
  · TodayAdventureBuilder 服务: 50/30/20 比例 + Boss 词序列
  · HomePage 增加"今日冒险"主入口卡片 + AppStorage[currentSessionPlan]
  · BattlePage 走 plan 分支启动今日冒险

Phase C — 奖励与结算 (可见反馈)
  · CoinAccount 服务 (魔法币、日上限 20、跨日重置)
  · WishlistStore 服务 (申请/等待/确认/已兑换状态机)
  · WishlistCatalog 内置 3 件兑换品
  · ParentLongPressGate 组件 (长按 3 秒确认)
  · WishlistPage 新页面
  · ResultPage 升级显示 coinsEarned / coinsTotal / dailyCapHit / 新学/复习
  · SessionResult 字段扩展
  · 测试与验收
```

### 1.4 关键不变量

- `BattleEngine.submitAnswer(option: string)` 签名不变 — 补字母题就是把"选中的字母"作为 option 传进来；引擎内按 `q.kind` 选 `option === q.answer` 还是 `option === q.letterAnswer`。
- `SessionResult` 仍由 `BattleEngine.buildSessionResult()` 产出；新字段（coinsEarned 等）由 `BattlePage.navigateToResult` 在路由前追加，与现有 `newlyLearnedCount` 同套路。
- `WrongAnswerStore` 持久化 schema 升级到 v2，但 `parseSnapshot` 兼容读 v1（v1 数据自动填新字段默认值）。
- `MonsterCatalog` 10 条目保留；自由练习/复习错题模式继续走 catalog（全部 Normal kind），行为 100% 兼容 V0.2。今日冒险走 `AdventureRegion.monsterPlan`，不走 catalog。

## 2. 数据模型

### 2.1 题型抽象 — `Question` 扩展

```text
enum QuestionKind { Choice = 'choice', FillLetter = 'fill-letter' }

class Question {
  // 现有 (保持原义)
  promptZh: string             // choice 题: 中文提示; fill-letter 题: 同样填中文释义
  answer: string               // 完整英文单词 (两类题都填)
  options: string[]            // choice 题: 3 个英文单词; fill-letter 题: []
  wordId: string

  // V0.3 新增
  kind: QuestionKind = QuestionKind.Choice  // 默认 Choice 保旧测试不变

  // fill-letter 题专用 (Choice 题留空)
  letterTemplate: string = ''   // 例 'a _ p l e', 单空格分隔
  missingIndex: number = -1     // 0-based, 缺哪一位
  letterOptions: string[] = []  // 长度 3, 候选字母 (lower-case 单字符)
  letterAnswer: string = ''     // 正确字母 (lower-case 单字符)
}
```

`isValid()` 按 `kind` 分派：
- `Choice`：原校验（`options.length === 3`、`answer ∈ options`、字段非空）。
- `FillLetter`：`letterOptions.length === 3`、`letterOptions` 全部一字符且互不重复、`letterOptions.indexOf(letterAnswer) >= 0`、`answer.toLowerCase()[missingIndex] === letterAnswer`、`letterTemplate` 非空。

## 3. 服务层

### 3.1 `FillLetterGenerator`（新）

独立服务，与 `QuestionGenerator` 平级。

```text
class FillLetterGenerator {
  constructor(rng?: RandomFn)
  generate(word: WordEntry, lastWordId?: string): Question | undefined
}
```

V0.3 初级生成规则：
1. 取 `word.word.toLowerCase()`，过滤非字母字符；得 `letters: string[]`。
2. 若 `letters.length < 3` → 返回 `undefined`（短词降级，调用方退回 Choice）。
3. `missingIndex` 从 `[1, letters.length - 1]` 区间随机（避开开头第 0 位）。
4. `letterAnswer = letters[missingIndex]`。
5. 干扰字母：在 `'abcdefghijklmnopqrstuvwxyz'` 中无放回随机抽 2 个，排除 `letterAnswer` 与该词内已出现的其他字母（避免歧义）。
6. `letterTemplate`：单空格分隔，缺失位渲染 `_`。例：`'a _ p l e'`。
7. `letterOptions` 三选一打乱。
8. 写入 `Question { kind: FillLetter, answer: word.word, wordId: word.id, ... }`。

### 3.2 `QuestionGenerator` 不变 + 引入 `IQuestionSource`

为了让 `BattleEngine` 可以接受两种生成器（`QuestionGenerator` 与 `PlanQuestionSource`），把 BattleEngine 内部对生成器的引用从具体类换成接口：

```text
interface IQuestionSource {
  nextQuestion(lastWordId?: string): Question
}
```

- `QuestionGenerator` 现有签名 `nextQuestion(lastWordId?: string): Question` 已经天然实现该接口（只产 Choice 题）。
- 不在 `QuestionGenerator` 上加 `prefer` 参数 — 自由练习/复习错题模式 V0.3 仍只出 Choice 题。FillLetter 题型只通过 `PlanQuestionSource` 产生。
- `BattleEngine` 构造函数 `generator: QuestionGenerator` 收紧为 `generator: IQuestionSource`，保持现有调用方不动（QuestionGenerator 满足接口）。

### 3.3 `MemoryScheduler`（新）

```text
class MemoryScheduler {
  applyAnswer(stat: WordStat, correct: boolean, nowMs: number): void
  classify(stat: WordStat, nowMs: number): MemoryState
}
```

状态推进表：

| 当前 state | 答对                                                   | 答错                              |
|-----------|--------------------------------------------------------|----------------------------------|
| New       | → Learning；`nextReviewMs = +1 天`                     | → Review；`+30 分钟`              |
| Learning  | 累计 correctCount ≥ 2 → Familiar (`+3 天`)；否则 (`+1 天`) | → Review；`+30 分钟`              |
| Review    | → Learning (`+1 天`)                                   | 保持 Review；`+30 分钟`            |
| Familiar  | 累计 correctCount ≥ 5 → Mastered (`+14 天`)；否则 (`+3 天`) | → Review；`+30 分钟`              |
| Mastered  | 保持 (`+14 天`)                                        | → Review；`+30 分钟`              |

间隔常量：
- `INTERVAL_30_MIN_MS = 30 * 60 * 1000`
- `INTERVAL_1_DAY_MS = 24 * 3600 * 1000`
- `INTERVAL_3_DAY_MS = 3 * INTERVAL_1_DAY_MS`
- `INTERVAL_14_DAY_MS = 14 * INTERVAL_1_DAY_MS`

`classify` 在出题时用，做"到期检查"：即使 stat 字段说 `Familiar` 或 `Mastered`，但 `nextReviewMs > 0 && nextReviewMs <= now` 时返回 `Review`，让到期词浮出。

`MemoryScheduler` 是无状态服务（仅依赖入参），便于单测固定 nowMs。

### 3.4 `TodayAdventureBuilder`（新）

```text
class TodayAdventureBuilder {
  constructor(
    repo: WordRepository,
    recorder: LearningRecorder,
    scheduler: MemoryScheduler,
    rng?: RandomFn
  )
  build(region: AdventureRegion, isFirstToday: boolean, nowMs: number): TodaySessionPlan
}
```

构建流程：
1. 总词槽数 `= region.monsterPlan.slots.length × 2`（默认 10）。
2. 把 `repo.all()` 按 `region.themeWordCategories` 加权：主题词权重 ×2，其他词权重 ×1。
3. `classify` 每个候选词为 review / learning / new：
   - `stat === undefined || memoryState === New` → `new`
   - `nextReviewMs > 0 && nextReviewMs <= now` 或 `memoryState === Review` → `review`
   - 其他 → `learning`
4. 按 50/30/20 比例无放回随机抽取（10 槽：5 review / 3 learning / 2 new；权重在抽样时影响概率）。
5. 任一池为空时，缺额按"review > learning > new"递补；若总词数 < 5 → 抛 `BuildError`，BattlePage 降级到普通模式。
6. `bossWordIds`：本局已抽中的 wordId 中按 review > learning > new 排序，取前 5 个，Boss 题时按 plan 顺序消费。
7. `isFirstToday` 由调用方（BattlePage 在 `aboutToAppear` 中先 `coinAccount.beginToday(now)` 再读 `snapshot().todayAdventureCompleted` 取反）传入。

### 3.5 `PlanQuestionSource`（新，实现 IQuestionSource）

```text
class PlanQuestionSource implements IQuestionSource {
  constructor(
    plan: TodaySessionPlan,
    repo: WordRepository,
    fillGen: FillLetterGenerator,
    choiceGen: QuestionGenerator,
    rng?: RandomFn
  )

  /** 必须由调用方在 BattleEngine 构造完成后注入；返回 1-based monsterIndex */
  setMonsterIndexProvider(provider: () => number): void

  nextQuestion(lastWordId?: string): Question
}
```

**接线方式**（避免循环构造）：

```text
const source = new PlanQuestionSource(plan, repo, fillGen, choiceGen, rng)
const engine = new BattleEngine(source, bc)
source.setMonsterIndexProvider((): number => engine.getState().monsterIndex)
engine.start()   // 第一次 nextQuestion 此时 monsterIndex = 1 → slot 0
```

`BattleEngine.submitAnswer` 在打死怪并 `monsterIndex += 1` 之后才调用 `nextQuestion`，因此 PlanQuestionSource 通过 provider 读到的总是"当前应该出题的怪"对应的 1-based 索引。无需 BattlePage 显式 advance。

**调度规则**（每次 `nextQuestion`）：
1. `slotIdx0 = clamp(provider() - 1, 0, plan.monsterSlots.length - 1)`；`slot = plan.monsterSlots[slotIdx0]`。
2. 决定 `kind`：
   - `Normal` → Choice
   - `Spelling` → FillLetter（短词降级 Choice）
   - `Review` → Choice（V0.3：题型默认 Choice，词来源仍是 plan 里 review 槽位）
   - `Elite` → 50/50 随机
   - `Boss` → 50/50 随机；wordId 优先从 `plan.bossWordIds.shift()` 取
3. 取 `wordId`：
   - `kind === Boss` 且 `bossWordIds` 非空 → 用 `bossWordIds.shift()`
   - 否则按内部 `wordCursor` 取 `plan.wordPlan[wordCursor]`，并跳过等于 `lastWordId` 的槽位（防同词连续），找到后 `wordCursor` 推进到该位之后
4. `repo.findById(wordId)`：
   - 找到 → 喂 `fillGen.generate(entry, lastWordId)`（FillLetter）或 `choiceGen` 单条产出（Choice，详见下）
   - 找不到（被改过词包等）→ 走 choiceGen 兜底
5. `wordCursor` 用尽（答错过多导致 wordPlan 耗光） → 用 choiceGen 兜底从 repo 抽（与 V0.2 同行为）。

**Choice 题用 PlanQuestionSource 产生时**：复用 `QuestionGenerator` 的内部生成逻辑，但对 wordId 锁定。在 `QuestionGenerator` 上新增辅助 `nextQuestionForWord(entry: WordEntry, lastWordId?: string): Question`，把答案锁定为 `entry`，干扰项仍从 repo 同类目抽。该辅助同时被 `PlanQuestionSource` 和现有 `nextQuestion` 共用。

### 3.6 `CoinAccount`（新）

```text
class CoinAccount {
  init(ctx: UIAbilityContext): Promise<void>
  beginToday(nowMs: number): void               // 跨日检测, 重置 today*
  earn(reason: string, amount: number, nowMs: number): number  // 返回实际入账
  redeem(wishId: string, cost: number): boolean
  snapshot(): CoinSnapshot                      // 只读副本 (深拷贝 txns)
  flushNow(): Promise<void>
}
```

不变量：
- `DAILY_CAP = 20`、`MAX_TXN_HISTORY = 50`。
- `formatDayKeyLocal(nowMs)` 纯函数：`new Date(nowMs)` 取本地年月日拼 `YYYY-MM-DD`。
- `beginToday`：当 `formatDayKeyLocal(now) !== todayDayKey` → 重置 `todayCoinsEarned = 0`、`todayAdventureCompleted = false`、`todayDayKey = formatDayKeyLocal(now)`。
- `earn(reason, amount)`：`actual = max(0, min(amount, DAILY_CAP - todayCoinsEarned))`；actual = 0 时不写流水；否则 `totalCoins += actual; todayCoinsEarned += actual`，push txn，超出 `MAX_TXN_HISTORY` 时丢首条。
- `earn` 的 `reason === 'today-first'` 时附加：`todayAdventureCompleted = true`（即使 actual 因封顶为 0，也置位）。
- `redeem(wishId, cost)`：`(totalCoins - spentCoins) >= cost` → `spentCoins += cost`，push 负 delta txn `redeem:<wishId>`，返回 true。日上限只约束 earn，不约束 redeem。
- 持久化 key：`wordmagic_coins`，全 JSON 序列化。

### 3.7 `WishlistStore`（新）

```text
class WishlistStore {
  init(ctx: UIAbilityContext): Promise<void>
  list(): MagicWish[]                           // 合并 catalog 默认 + 持久化 state
  request(wishId: string, account: CoinAccount, nowMs: number): boolean
                                                // Idle → Pending; 检查可用余额 ≥ cost
  confirmByParent(wishId: string, account: CoinAccount, nowMs: number): boolean
                                                // Pending → Confirmed + account.redeem
  cancel(wishId: string): void                  // Pending → Idle (V0.3 隐藏入口, 暂不上 UI)
  acknowledge(wishId: string): void             // Confirmed → Idle
  flushNow(): Promise<void>
}
```

不变量：
- `request` 不扣币；只检查"可用余额 = totalCoins - spentCoins ≥ cost"，否则返回 false。
- `confirmByParent` 内部：先 `account.redeem` 成功才把 wish.state 推到 Confirmed，时序保证币没扣成功时状态不前进。
- `acknowledge` 让 UI 在显示完"已兑换"提示后把 wish 状态重置为 Idle，可重复申请。
- 持久化 key：`wordmagic_wishlist`。

```text
  4. source = new PlanQuestionSource(plan, repo, fillGen, choiceGen)
  5. engine = new BattleEngine(source, bc); engine.start()
  6. 战斗结束 → engine.buildSessionResult() → BattlePage 累计 coinAccount.earn 多个 reason
     → 写入 SessionResult 新字段 → 路由到 ResultPage
```

`ParentLongPressGate` 实现：
- 圆形按钮 + 圆环进度。
- `onTouch(Down)` 启动 `setInterval`（每 33 ms 推进 progress += 33/3000）。

`onApply(wish)`：`store.request(wish.id, account, now)` → 失败 toast"魔法币不够"。

`onParentConfirm(wish)`：`store.confirmByParent(wish.id, account, now)` → 成功后切到 Confirmed 状态、更新余额浮标、计时 1.5 秒后 `acknowledge`。

## 6. 持久化、迁移与降级

### 6.1 Preferences key 一览

| key                       | 内容                            | 版本   |
|--------------------------|--------------------------------|--------|
| `wordmagic_learning`     | LearningSnapshot (stats[])      | v1 → v2 |
| `wordmagic_coins`        | CoinSnapshot                    | v1     |
| `wordmagic_wishlist`     | WishlistSnapshot                | v1     |

`AppStorage` 临时 key：`currentSessionPlan`（路由中转，进入 BattlePage 后由 `navigateToResult` 删除）。

### 6.2 Schema v1 → v2 迁移

`parseSnapshot` 在 `version` 缺失或 `=== 1` 时，对每个 `WordStat` 自动填：
- `lastCorrectMs = correctCount > 0 ? lastAnsweredMs : 0`
- `nextReviewMs = 0`
- `memoryState = correctCount > 0 ? Learning : New`

写回时统一 `version = 2`。下次 `recordAnswer` 由 `MemoryScheduler.applyAnswer` 自然推进。

### 6.3 失败降级

| 失败场景                           | 降级行为                                                       |
|----------------------------------|---------------------------------------------------------------|
| `loadFromRawfile` 失败            | 沿用现有 "Failed to load words" 路径                            |
| `TodayAdventureBuilder.build` 词池过小 | 抛 `BuildError`；HomePage 今日冒险卡灰态显示 "暂无可用题库"        |
| `FillLetterGenerator.generate` 短词降级 | PlanQuestionSource 自动退回 Choice，不抛错                       |
| `CoinAccount.init` / `WishlistStore.init` 失败 | UI 显示余额 0、申请按钮 disabled；战斗本身不阻塞 |
| `parseSnapshot` 失败             | 沿用现有"返回空 snapshot"路径                                   |
