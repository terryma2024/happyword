## 1. 范围与架构概览

### 1.1 V0.3.9 一句话

> 把魔法愿望单的兑换流程从「家长长按 3 秒环」升级为「6 位 PIN 弹窗 + GiftBox 庆祝动画 + 持久化兑换记录页」，同时把卡片版式改成「左侧名+价、右侧奖品 emoji + 申请兑换按钮并排靠右」，wishlist 头部新增 📜 入口进兑换记录页。

### 1.3 关键不变量（必须保持）

- `MagicWish` 类形状不变；`WishState.Pending` 枚举值保留（reader 看到旧 Pending → 归一为 Idle）。
- `WishlistStore.list()` 返回的 `MagicWish[]` 形状不变；新增的 redeemWithPin 不破坏既有 init / list / acknowledge / cancel 接口。
- `CoinAccount.redeem(wishId, cost)` 完全不动，仍是单笔账户记账唯一入口。
- `GiftBox` 组件本身**完全不动**——既有 trigger-driven 行为已经满足"播完 1.5s 自闭"，外层 modal 用 hold + unmount 控制额外 1.5s。
- `ParentLongPressGate` 文件保留（避免删 file 影响其它潜在依赖），但 wishlist 不再 import 它。
- `GameConfig` 序列化向后兼容：旧无 `parentPin` 字段的 snapshot 反序列化为 `parentPin: ''`（空串 = 未设置）。
- `ConfigPage` 现有 5 个调节项（玩家血量 / 怪物血量 / 怪物数量 / 倒计时 / 词库）行为完全不变；只在 categoryRow 上方/下方插入「家长密码」一行。
- 路由 push / back 行为遵循现有 `router.pushUrl({ url })` 模式；新增页都需在 `main_pages.json` 注册。
- ArkTS `@Builder` 反应式约束（v0.3.6 letter-spacing 教训 + 本分支 precursor commit 修的同源问题）继续遵守——所有 ForEach 迭代器内**不再调用**参数化 @Builder。

## 4. 状态机简化（S3）

### 4.1 新流程

```
Idle ──[tap 申请兑换]──▶ PIN dialog 开启
                          │
                          ├──[输错]──▶ 抖动+清空，留在 dialog
                          ├──[取消/遮罩外]──▶ 回 Idle, dialog 关
                          └──[输对]──▶ dialog dismiss
                                       │
                                       ▼
                          CoinAccount.redeem(id, cost) [失败概率极低，列入兜底]
                                       │ ok
                                       ▼
                          RedemptionHistoryStore.add(record)
                          MagicWish.state = Confirmed
                          this.showGiftBox = true
                          this.giftBoxTrigger += 1   (重启 GiftBox 内部时间轴)
                          ▼
                          GiftBox 自播 ~1680ms
                          ▼
                          setTimeout 1500ms → showGiftBox = false
                          ▼
                          卡片显示 "已兑换 ✓" 1500ms (复用现有 ACK_DELAY_MS)
                          ▼
                          MagicWish.state = Idle
```

### 4.2 关键时间常量

```ts
// WishlistPage.ets
const GIFTBOX_HOLD_AFTER_PLAY_MS: number = 1500;  // 播完后再停留
const ACK_DELAY_MS: number = 1500;                // 复用现有
// → 总从 redeem 成功到 wish 回 Idle ≈ 1680 + 1500 + 1500 ≈ 4680ms
```

### 4.3 状态保护

- GiftBox 播放期间，`hitTestBehavior(HitTestMode.Block)` 拦截所有底层点击。
- 同一时刻只允许一个 GiftBox 播放（`@State private showGiftBox: boolean`），重复触发被守卫拒。
- 「← 返回」在 GiftBox 期间被遮罩挡住，无法退出页（避免兑换链断裂）。`onBackPress` 在 `showGiftBox === true` 时返回 `true` 拦截系统返回手势。

属性 / 状态：

```ts

  controller?: CustomDialogController
  storedPin: string = ''                  // 父页传入
  onCorrect: () => void = () => {}
  onCancel: () => void = () => {}

  @State private input: string = ''       // 0-6 chars
```

### 6.2 行为表

| 事件 | 反应 |
| --- | --- |
| 点 0-9 | `input += digit`；若 `input.length === 6` → `validate()` |
| 点 ⌫ | `input = input.slice(0, -1)`；隐藏 errorVisible |
| 点 取消 | controller.close()，触发 onCancel |
| 点遮罩外 | 同上（ArkUI CustomDialog 默认行为） |
| validate 输对 | controller.close()，触发 onCorrect |
| validate 输错 | `shake++`（驱动 -8 / +8 / 0 三段位移动画）+ `errorVisible = true` + `input = ''` |
| 输错任意点击 | errorVisible = false |

### 6.3 抖动动画

```ts
// shake 自增触发 @Watch
@Watch('onShake') shake: number = 0
@State private shakeOffset: number = 0

private onShake(): void {
  this.getUIContext().animateTo({ duration: 60, curve: Curve.Linear }, () => {
    this.shakeOffset = -8;
  });
  setTimeout(() => {
    this.getUIContext().animateTo({ duration: 60, curve: Curve.Linear }, () => {
      this.shakeOffset = 8;
    });
  }, 60);
  setTimeout(() => {
    this.getUIContext().animateTo({ duration: 60, curve: Curve.Linear }, () => {
      this.shakeOffset = 0;
    });
  }, 120);
}
```

### 6.4 纯逻辑可测点

抽出 `validatePin(input: string, stored: string): boolean`，纯函数，单测 ≥3 case：

- 输入 == stored → true
- 输入 != stored → false
- 长度不一致 → false

`pushDigit(buf: string, d: string): string`、`popDigit(buf: string): string` 同样抽纯函数。

### 7.2 触发与销毁

```ts
@State private showGiftBox: boolean = false
@State private giftBoxTrigger: number = 0

private onPinCorrect(wishId: string, cost: number): void {
  const ok: boolean = this.coins.redeem(wishId, cost);
  if (!ok) {
    // CoinAccount.redeem 极少失败（已校验 balance），但作为防御
    console.warn(`Redeem rejected for ${wishId}`);
    return;
  }
  this.history.add(makeRecord(wishId, cost, this.findWish(wishId)));
  this.store.markConfirmed(wishId);   // 新方法，仅置 state=Confirmed + confirmedAt
  this.showGiftBox = true;
  this.giftBoxTrigger += 1;
  this.refresh();

  setTimeout(() => {
    this.showGiftBox = false;
    setTimeout(() => {
      this.store.acknowledge(wishId);
      this.refresh();
    }, ACK_DELAY_MS);
  }, GIFTBOX_INTRINSIC_MS + GIFTBOX_HOLD_AFTER_PLAY_MS);
  // 1680 + 1500 = 3180ms
}
```

`GIFTBOX_INTRINSIC_MS` 定义：跟 GiftBox 内部 `AUTO_CLOSE_DELAY_MS (1500)` + 盖子合上动画 180ms = `1680`。**写在 WishlistPage 文件顶部并加注释指向 GiftBox.ets 的常量来源**——避免漂移。

## 8. 兑换记录 store + 页（S7）

### 8.1 RedemptionRecord 模型

```ts
// models/RedemptionRecord.ets
export class RedemptionRecord {
  id: string = ''           // crypto-style: ts-millis + random suffix
  ts: number = 0            // Date.now()
  wishId: string = ''
  displayName: string = ''
  iconEmoji: string = ''
  costCoins: number = 0
}

export class RedemptionHistorySnapshot {
  version: number = 1
  records: RedemptionRecord[] = []
}

export const REDEMPTION_PREFS_NAME: string = 'wordmagic_redemption_history'
export const REDEMPTION_PREFS_KEY: string = 'snapshot_v1'
export const REDEMPTION_HISTORY_CAP: number = 50
```

### 8.2 RedemptionHistoryStore 服务

完全 mirror `WishlistStore` 的形状：

- `init(ctx)` / `injectPreferencesForTest(p)` / `flushNow()`
- `add(r: RedemptionRecord): void` — push tail，超过 `REDEMPTION_HISTORY_CAP` 丢头
- `list(): RedemptionRecord[]` — 返回 deep clone（防外部突变）；按 `ts` desc 排序后返回
- `__resetRedemptionHistoryStoreForTest()` 单例 reset

空态：标题下 `Text('还没有兑换记录').fontColor('#888888')`，居中。

### 8.4 跨页一致性

- 兑换成功落盘后 wishlist refresh；用户随即进入历史页应能看到这一条（store 是单例，无需 push 信号）。
- 历史页 `onPageShow` 重新 `list()` 一次，避免回程时数据不更新。

## 9. 错误处理

| 场景 | 行为 |
| --- | --- |
| 未设 PIN，点申请兑换 | 弹 AlertDialog «家长密码尚未设置» / «请先在「⚙ 设置」→「家长密码」中设置 6 位数字» / [知道了]，不打开 PIN dialog |
| 余额不足 | 不渲染按钮，渲染「还差 N ✨」（不会发生 onClick） |
| PIN 输错 | 抖动 + 清空 + 红字 «密码不正确，请重试»，无锁定无计数 |
| CoinAccount.redeem 返回 false（理论上不会，因 balance 已校验） | console.warn + 不开 GiftBox + dialog 关闭 + 留在 Idle |
| GiftBox 期间用户拔电 / 后台 | next aboutToAppear / onPageShow 时 store init 重新拉 snapshot；wishes 中 Confirmed 视情况：本期 spec 决策 = onPageShow 检测到任意 wish.state==Confirmed 立刻 acknowledge → Idle（避免卡死） |
| 历史 store cap 溢出 | shift() 丢最早一条，不报错 |
| preferences 写失败 | 现有 `console.error` 一致行为，不阻塞 UI |

## 11. 已知风险

| 风险 | 缓解 |
| --- | --- |
| ArkUI CustomDialog 默认动画与 GiftBox modal 在同一 Stack 重叠时 z-order 冲突 | PIN dialog 由 CustomDialogController 管理，先关闭再 setShowGiftBox(true)；不会同时在屏幕上 |
| HitTestMode.Block 在某些 ArkUI 版本对 onBackPress 不生效 | 显式实现 `onBackPress(): boolean { return this.showGiftBox; }` 兜底 |
| 6 位数字键盘 12 个 Button id 全部唯一 | id 模板 `PinKey0` ~ `PinKey9` + `PinKeyDel` + `PinKeyCancel` |
| 兑换页期间换日 / 时区切换导致 ts 与 formatLocalTimestamp 输出不一致 | ts 用 `Date.now()` 写死、format 在读取时本地化；不解决跨时区一致问题（单设备应用，可接受） |
| 旧持久化里 state='pending' 的 wish 升级到 v0.3.9 会被静默归一为 Idle | overall_spec § 9.5 添加迁移说明；reader 内 `coerceState('pending') → Idle`（覆盖原行为）|
