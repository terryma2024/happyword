## 1. 范围与架构概览

### 1.1 V0.3.10 一句话

> 用 Recraft V4 vector 给游戏生成一套统一的「童话魔法羊皮纸」风格 UI 图标——重做 app 启动图标（前景 magician + 背景魔法夜空 + startIcon）和首页 5 个圆形按钮（📚📖🪄⚙ 工具栏 + 愿望单页头部 📜），让全 app 视觉从「emoji 拼贴」升级为「自家美术资产」，并保持现有 boss SVG 的同源调性（暖色羊皮纸、柔光、童话不恐怖）。

### 1.4 关键不变量（必须保持）

- 5 个被替换的 Button（HomeReview / HomeCodex / HomeWishlist / HomeConfig / WishlistHistory）的 **id 不变**、**width / height / backgroundColor / borderRadius 不变**、**Type=Circle 不变**（Button 是容器，把内部「文本 emoji」换成「Image 子节点」即可）。

- App 启动图标 `$media:layered_image` JSON 不改（前景/背景文件名复用），所以 `module.json5` / `app.json5` **零改动**。
- AppStorage / preferences key 全部不动（icon 重构不涉及任何 state）。
- 现有 V0.3.9 wishlist GiftBox modal / PIN dialog / RedemptionHistoryPage / 11+ UI 套件 0 回归。

## 2. 视觉规范（Visual Identity）

### 2.1 风格关键词

> **童话魔法羊皮纸（Warm Parchment Fairy-tale）**
>
> 平面着色 + 柔光高光，温暖羊皮纸 / 童话夜空配色，单一可读剪影，56dp 尺寸下仍清楚识别，与现有 boss SVG（witch / phoenix / unicorn 等）同源调性，不做立体渐变 / 不做反光高亮 / 不做 8-bit 像素风。

### 2.5 HarmonyOS 启动图标尺寸细则

HarmonyOS NEXT 的 layered icon 规范：

- 建议在 `resources/base/media` 下提供 `1024×1024` 的前景与背景 PNG，系统会自动按设备密度（sdpi/mdpi/ldpi/xldpi）缩放。
- `startIcon.png` 用作启动 splash window，建议 `216×216`，PNG 透明背景。
- 不需要为 launcher 图标主动建 `resources/sdpi/media` 等密度限定目录（系统已能从 1024 自适应，多目录反而提高维护成本）；本期采用单源 1024 + startIcon 216 的简化分发策略。

## 3. 资产生成流程

### 3.1 SVG 生成（Recraft V4 vector）

`tools/recraft/generate-icons.sh` 模板（mirror `generate-bosses.sh`）：

```bash
TEMPLATE_TAIL="original cute fairy-tale magic UI icon for a children's English vocabulary adventure game, single subject centred in a square frame, simple readable silhouette suitable for 56-pixel rendering, flat shading with soft highlights, warm parchment fairy-tale palette: deep red E63946, warm gold FFB400, navy ink 1D3557, cream parchment FFF8E7, soft pink FCEAEA, plain transparent background, no text, no copyrighted character likeness, gentle whimsy mood not scary"

ENTRIES=(
  "foreground|...subject..."
  "background|...subject..."
  "startIcon|...subject..."
  "review|...subject..."
  "codex|...subject..."
  "wand|...subject..."
  "gear|...subject..."
  "scroll|...subject..."
)
```

输出：

```text
generated/recraft/icons/<name>.svg     # 真正的 Recraft V4 vector 输出（保留作可复现证据）
generated/recraft/icons/<name>.json    # API 响应原文，含 image_id / url
```

skip-if-exists：脚本检测目标 SVG `> 5000` bytes 即跳过该条（与 `generate-bosses.sh` 一致），避免误覆盖。

### 3.2 PNG 光栅化（rsvg-convert）

`tools/recraft/svg-to-png.mjs`：薄包装 librsvg `rsvg-convert`，用法：

```bash
node tools/recraft/svg-to-png.mjs --in generated/recraft/icons/foreground.svg --out entry/src/main/resources/base/media/foreground.png --size 1024
node tools/recraft/svg-to-png.mjs --in generated/recraft/icons/background.svg --out entry/src/main/resources/base/media/background.png --size 1024
node tools/recraft/svg-to-png.mjs --in generated/recraft/icons/startIcon.svg --out entry/src/main/resources/base/media/startIcon.png --size 216
```

`tools/recraft/icons-to-launcher.sh` 把这三条命令封装成一个执行入口（脚本中调用 `node svg-to-png.mjs ...` 三次 + 简单错误返回码聚合）。

### 3.3 SVG 同时落到 rawfile

5 个 toolbar/scroll SVG 直接 `cp generated/recraft/icons/{review,codex,wand,gear,scroll}.svg → entry/src/main/resources/rawfile/icons/`，无需光栅化（在 ArkUI 里通过 `Image($rawfile('icons/foo.svg'))` 渲染，HarmonyOS 内置 SVG renderer 已足够）。

`generate-icons.sh` 末尾追加一个「sync to rawfile」段，把这 5 个 SVG 自动复制到位（避免手工 cp）。

## 4. ArkTS 集成方案

### 4.1 Image 子节点替换 emoji 文本

HarmonyOS ArkTS Button 支持把 `Image` 作为子节点替换文本（参考官方 ButtonAttribute API）。统一改造：

```typescript
// Before (v0.3.9):
Button('📚')
  .id('HomeReviewButton')
  .type(ButtonType.Circle)
  .width(56).height(56)
  .fontSize(24)
  .fontColor(this.reviewEnabled() ? '#E63946' : '#999999')
  .backgroundColor(this.reviewEnabled() ? '#FCEAEA' : '#EEEEEE')
  .onClick((): void => { ... });

// After (v0.3.10):
Button({ type: ButtonType.Circle, stateEffect: true }) {
  Image($rawfile('icons/review.svg'))
    .width(28).height(28)
    .objectFit(ImageFit.Contain)
    .opacity(this.reviewEnabled() ? 1.0 : 0.4);
}
  .id('HomeReviewButton')
  .width(56).height(56)
  .backgroundColor(this.reviewEnabled() ? '#FCEAEA' : '#EEEEEE')
  .onClick((): void => { ... });
```

- `Button` 用「容器形式」`Button() { Image(...) }`，里面只放一个 `Image`。
- 大小尺寸：按钮 56×56，里面图标 28×28（≈ 50% 内填充），保留按钮自带的圆角与 stateEffect ripple。
- Lock 状态：`opacity(0.4)` 等效原来的 fontColor 灰化效果（背景色仍是 `#EEEEEE`）。
- WishlistHistoryButton（40×36）→ Image 18×18。
- 4 个 HomeToolbar 按钮里面 Image 28×28（除了 ⚙ 它原 fontSize 28，可考虑 30×30 让齿轮纹路更清楚）。

## 7. 风险与回滚


| 风险                                               | 应对                                                                                                                                             |
| ------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| Recraft V4 vector 单次生成质量参差（颜色偏离 / 主体偏小 / 包含意外文字） | 每个 icon 视情况手动跑 1-3 次，只把最佳那次同步到 rawfile；保留 generated/recraft/icons/.json 留对比记录                                                                  |
| ArkTS Button 容器子节点 ripple / 圆角失真                 | 容器写法是官方 sample 推荐；如果出问题 fallback 到 Stack(Button + 重叠 Image) 模式（多一层视图，不是 blocker）                                                               |
| HarmonyOS SVG renderer 不支持某些复杂 path              | rawfile SVG 在 boss 章节已成功使用（witch.svg 65KB / phoenix.svg 780KB 都正常），icon 体量更小，风险低；如个别 SVG 打不开就给该图标手动 rsvg-convert 出 PNG fallback 替换（资源路径仅文件后缀变） |
| launcher PNG 替换后旧设备缓存图标                          | 多数 OEM 桌面会自动更新；最坏情况让用户重启或重新安装 hap                                                                                                              |
