## 1. 范围与架构概览

### 1.1 V0.3.7 一句话

> 在首页顶部工具栏新增 📖 入口，进入"怪物图鉴页"，**只展示 3 张**对应美术资源的怪物档案（Slime / Zombie / Dragon），左右双箭头切换、边界禁用，每张档案给出 3-4 句中文世界观文案，**不提任何玩法、不接音频、不持久化索引**。

> **范围收窄说明**：roadmap §7 原本描述"现有 10 个怪物"全部进入图鉴；brainstorming 阶段确认实际美术资源只有 3 套（slime/zombie/dragon SVG），V0.2 catalog 的 10 条 fallback 同一个 slime 资产，进入图鉴会出现 10 张几乎相同的卡片。因此本 spec 把展示对象收窄为 3 套真实美术对应的"难度形象"，每张承担一个 `MonsterKind` 桶位（普通 / 拼写·复习 / 精英·Boss）。后续若新增美术，再扩 `MONSTER_CODEX` 数组即可，无需改其他模块。

|------|---------------------|----------------------------------------|--------------------------------------------|
| 1    | `HomeCoinBalance`   | 金色胶囊，`✨ ${coinBalance}`           | 不变                                       |

| **3**| **`HomeCodexButton`** | **56×56 圆形，📖，红字 `#E63946` / `#FCEAEA` 底** | **`router.pushUrl('pages/MonsterCodexPage')`** |
| 4    | `HomeWishlistButton`| 56×56 圆形，🪄                         | 不变                                       |

### 2.2 图鉴页布局

垂直单列堆叠，居中对齐：

```text
┌───────────────────────────────────────────┐
│ [← Back]            怪物图鉴               │  ← 顶栏 (CodexBackButton + 标题)
├───────────────────────────────────────────┤
│                                           │
│        ⬅      [   头像   ]      ➡         │  ← Row: CodexPrev + CodexAvatar + CodexNext
│                                           │
│                  Slime                    │  ← CodexName
│                                           │
│                「普通怪物」                │  ← CodexKindLabel (灰底胶囊)
│                                           │
│      Slime 是一团从晨露中诞生的轻盈…       │  ← CodexDescription (居中, padding 24vp)
│                                           │
│                  1 / 3                    │  ← CodexPositionIndicator
│                                           │
└───────────────────────────────────────────┘
```

**首次进入永远停在 index 0（Slime）**——无 AppStorage、无类静态变量。
**边界状态**：`currentIndex === 0` 时 `CodexPrev.enabled(false)` + 灰色；`currentIndex === MONSTER_CODEX.length - 1` 时 `CodexNext.enabled(false)` + 灰色。
**手势**：仅左右箭头点击，**不**做 swipe / 长按 / 缩放。
**音频**：无（不调用 TTS、不播放任何音效）。
**返回**：`CodexBackButton` 调用 `router.back()`，HomePage 状态（金币、复习计数等）不变。

## 3. 数据模型

### 3.1 `data/MonsterCodex.ets`

```typescript
export class CodexEntry {
  key: string = '';
  nameEn: string = '';
  kindLabelZh: string = '';
  descriptionZh: string = '';
  assetPath: string = '';
}

function makeCodexEntry(
  key: string, nameEn: string, kindLabelZh: string,
  descriptionZh: string, assetPath: string
): CodexEntry {
  const e: CodexEntry = new CodexEntry();
  e.key = key;
  e.nameEn = nameEn;
  e.kindLabelZh = kindLabelZh;
  e.descriptionZh = descriptionZh;
  e.assetPath = assetPath;
  return e;
}

export const MONSTER_CODEX: CodexEntry[] = [
  makeCodexEntry('slime',  'Slime',  '普通怪物',   /* lore */, 'character/slime.svg'),
  makeCodexEntry('zombie', 'Zombie', '拼写专家',   /* lore */, 'character/zombie.svg'),
  makeCodexEntry('dragon', 'Dragon', '精英挑战者', /* lore */, 'character/dragon.svg'),
];
```

注释中的 `/* lore */` 在实现时填入 §2.3 的中文文案原文。

### 3.2 索引语义

- 数组下标 `0..MONSTER_CODEX.length - 1` 对外稳定；UI 测试可用 `1 / N` 这种 1-based 形式展示。
- `assetPath` 在 codex 内**写死字符串**，**不**调用 `characterAssetForKind`——避免 codex 顺序与战斗 kind 映射纠缠。两边 SVG 路径一致性由 §5.1 单测断言保护。
- 不导出工具方法（`getCodexByKey` 等）—— V0.3.7 不需要。

## 4. UI 实现要点

### 4.1 `pages/MonsterCodexPage.ets`

```typescript
@Entry @Component
export struct MonsterCodexPage {
  @State private currentIndex: number = 0;

  private hasPrev(): boolean { return this.currentIndex > 0; }
  private hasNext(): boolean { return this.currentIndex < MONSTER_CODEX.length - 1; }
  private current(): CodexEntry { return MONSTER_CODEX[this.currentIndex]; }

  private goPrev(): void { if (this.hasPrev()) { this.currentIndex -= 1; } }
  private goNext(): void { if (this.hasNext()) { this.currentIndex += 1; } }

  build() {
    Column() {
      this.topBar();
      this.avatarRow();
      this.nameBlock();
      this.descriptionBlock();
      this.positionIndicator();
    }
    .width('100%').height('100%')
    .padding({ top: 32, bottom: 32 });
  }

  @Builder private topBar() { /* ← Back 按钮 + "怪物图鉴" 标题 */ }
  @Builder private avatarRow() { /* CodexPrev / CodexAvatar (Image $rawfile) / CodexNext */ }
  @Builder private nameBlock() { /* CodexName + CodexKindLabel */ }
  @Builder private descriptionBlock() { /* CodexDescription, padding 24vp, 行高 22vp */ }
  @Builder private positionIndicator() { /* `${idx+1} / ${MONSTER_CODEX.length}` */ }
}
```

### 4.2 反应式约束

所有 `@Builder` 体内**只用** `this.currentIndex`、`this.current()`、`this.hasPrev()`、`this.hasNext()`，**不**接受参数。这是 v0.3.6 字母间距修复时踩过的坑：参数化 `@Builder` 在 ArkUI 不会跟踪 `@State` 依赖、不会重渲染。

### 4.3 视觉细节

- `CodexAvatar` Stack：`width(160).height(160)`，背景圆角卡片 `#F8F9FA`，圆角 24，`Image($rawfile(this.current().assetPath))` 占 128×128，`syncLoad(true)` 保证 UI 测试快照稳定。
- `CodexPrev` / `CodexNext`：`Button(ButtonType.Circle)`、56×56、字号 28；`enabled` 根据 `hasPrev() / hasNext()` 切换；`enabled === false` 时 `fontColor('#999999') backgroundColor('#EEEEEE')`，`true` 时 `fontColor('#E63946') backgroundColor('#FCEAEA')`（沿用 `HomeReviewButton` / `HomeWishlistButton` / `HomeCodexButton` 的红色配色，与图鉴入口 📖 视觉统一）。
- `CodexName`：`fontSize(22) fontWeight(FontWeight.Bold) fontColor('#1D3557')`。
- `CodexKindLabel`：胶囊，`#EAF2F8` 底、`#457B9D` 字、`fontSize(14) padding({ left: 12, right: 12, top: 4, bottom: 4 }) borderRadius(12)`。
- `CodexDescription`：`fontSize(14) lineHeight(22) fontColor('#333333') textAlign(TextAlign.Center) padding({ left: 24, right: 24 })`。
- `CodexPositionIndicator`：`fontSize(12) fontColor('#888888') margin({ top: 16 })`。

### 4.4 路由注册

`entry/src/main/resources/base/profile/main_pages.json` 在 `WishlistPage` 之后追加 `"pages/MonsterCodexPage"`。`HomePage` 顶栏插入：

```typescript
Button('📖')
  .id('HomeCodexButton')
  .type(ButtonType.Circle)
  .width(56).height(56)
  .fontSize(24)
  .fontColor('#E63946')
  .backgroundColor('#FCEAEA')
  .onClick((): void => {
    router.pushUrl({ url: 'pages/MonsterCodexPage' }).catch((err: BusinessError): void => {
      console.error(`HomePage: pushUrl MonsterCodexPage failed: ${JSON.stringify(err)}`);
    });
  });
```

插入位置：现有 `HomeReviewButton` 与 `HomeWishlistButton` 之间，工具栏 `Row({ space: 12 })` 不变。

## 5. 测试

### 5.1 单元测试 — `entry/src/test/MonsterCodex.test.ets`

注册到 `entry/src/test/List.test.ets`，5 类断言：

1. **数组长度**：`MONSTER_CODEX.length === 3`。
2. **字段非空**：每条 `key / nameEn / kindLabelZh / descriptionZh / assetPath` 长度均 > 0。
3. **唯一性**：`key` 集合等于 `{'slime', 'zombie', 'dragon'}`，无重复；`nameEn` 同样去重。
4. **文案长度**：`descriptionZh` 字符数 ∈ `[40, 200]`（中文 3-4 句的合理边界，防止误删半段或粘贴超长）。
5. **资产同步守卫**：
   - `MONSTER_CODEX[0].assetPath === characterAssetForKind(MonsterKind.Normal)`
   - `MONSTER_CODEX[1].assetPath === characterAssetForKind(MonsterKind.Spelling)`
   - `MONSTER_CODEX[2].assetPath === characterAssetForKind(MonsterKind.Elite)`

### 5.2 UI 测试 — `entry/src/ohosTest/ets/test/MonsterCodexFlow.ui.test.ets`

注册到 `entry/src/ohosTest/ets/test/List.test.ets` 末尾。每个用例之前用辅助函数 `returnToHome` 兜底（参考 `HomeToolbarLocked.ui.test.ets` 的同名 helper），保证从 HomePage 出发。

| 用例 id | 步骤 | 主要断言 |
|---|---|---|
| `homeCodexButtonOpensCodexPage` | HomePage → click `HomeCodexButton` | `CodexAvatar` / `CodexName('Slime')` / `CodexKindLabel('普通怪物')` / `CodexPositionIndicator('1 / 3')` 都存在；`CodexPrev` 不可点；`CodexNext` 可点 |
| `nextArrowAdvancesAndDisablesAtEnd` | 进入 codex → click `CodexNext` ×2 | `CodexName('Dragon')` / `CodexPositionIndicator('3 / 3')`；`CodexNext` 不可点；再 click `CodexNext`，名字仍为 `Dragon`（no-op） |
| `prevArrowGoesBackAndDisablesAtStart` | 进入 codex → click `CodexNext` ×2 → click `CodexPrev` ×2 | `CodexName('Slime')` / `CodexPositionIndicator('1 / 3')`；`CodexPrev` 不可点 |
| `backButtonReturnsToHome` | 进入 codex → click `CodexBackButton` | HomePage `HomeStartButton` 可见；`CodexAvatar` 不再存在 |

每次 `pushUrl` / `back` 之后等待 1500ms 让路由动画稳定（参考 v0.3.6 已经将所有进入 BattlePage 的等待时间统一调到 1500ms）。

### 5.3 测试稳定性

- 起始位置永远是 index 0（无持久化），用例之间天然独立。
- `MonsterCodexPage` 的 Image 用 `syncLoad(true)`，UI 测试不会因为图像异步加载而拿到空快照。
- 4 个用例运行总耗时预估 < 60s（每用例 ≤ 15s）。

## 6. 风险 & 缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| Image `$rawfile` 资源缺失 | codex 头像空白 | §5.1 资产同步单测；项目已有 slime/zombie/dragon SVG（见 `CharacterAssets.test.ets`）|
| ArkTS `@Builder` 反应式 | 切换 index 时 UI 不更新 | §4.2 强制 builder 内直接读 `this.` |
| HomePage toolbar 横向溢出 | 5 个圆形按钮 + 金币胶囊宽度超出 | 5×56 + 4×12 + 70 ≈ 398vp，phone 横屏宽 ~640vp，余量充足；codelinter & 真机回归会暴露任何溢出 |
| 现有 UI 套件回归 | 11 个套件中任一失败 | spec 不动既有 id；首页 toolbar 仅插入新按钮，不重排既有 id；执行 codelinter + ohosTest 全跑作为门禁 |
| codelinter `avoid-overusing-custom-component-check` | 编译告警 | 仅用 `@Builder`，不新增 `@Component`（§1.4）|
| 文案被误写为简体/繁体混合或半角标点 | 文学风格被破坏 | 所有 `descriptionZh` 用中文标点（"，。"），spec self-review 阶段抽查一遍 |

## 7. 非目标（V0.3.7 明确不做）

- 不增加新美术资源（不画 Lava Imp / Frost Wisp / Coral Slime 等单独头像）。
- 不让 V0.2 catalog 的 10 条进入 codex（codex 永远是 3 张）。
- 不接解锁 / 收藏 / 已遇见标记（默认全部可见）。
- 不做音频朗读（无 TTS、无音效）。
- 不做 swipe / 长按 / 缩放手势。
- 不持久化 `currentCodexIndex`（每次进入永远从 Slime 开始）。
- 不接服务端 / 远程文案。
- 不增加多语言切换（V0.3.7 中文档案文案，英文名固定）。
- 不改动战斗模型、Battle / Result / Config / Wishlist 任何已有页面。
