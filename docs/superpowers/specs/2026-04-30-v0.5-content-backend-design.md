# V0.5 内容后台与 LLM 题库版 — 总体设计

- **Date:** 2026-04-30
- **Roadmap:** [WordMagicGame_roadmap.md §12 V0.5](../../WordMagicGame_roadmap.md)
- **Status:** Design-for-implementation; no code changes in this spec step
- **Target version range:** V0.5.1 → V0.5.7（拆 7 个子版本）
- **Tech stack:** Python 3.12 + FastAPI + uv + MongoDB Atlas + Vercel Blob + OpenAI；HarmonyOS 客户端零新增 oh-package 依赖
- **Out-of-scope:** 家长账号 / 设备绑定（V0.6）、句子填词 UI / AI 剧情（V0.7）、BattleAudioMixer（V0.8）、Cocos2D 重写（V0.9）

---

## 1. 背景与一句话设计

### 1.1 问题背景

V0.4.x 已经把客户端学习内核做扎实（题型抽象、记忆状态、今日计划、学习报告、自定义愿望单、视觉与音效抛光）。下一阶段的最大瓶颈是**内容生产**：

- 词库扩展全靠手改 `entry/src/main/resources/rawfile/data/words_v1.json`，发版才生效；
- 题目干扰项、例句、插画、发音 mp3 都没有规模化生产路径；
- 没有审核流，AI 生成内容无法在不接触儿童端的前提下迭代；
- 无法做按主题（学校 / 海洋 / 家庭）的快速课包搭建。

V0.5 的目标是上线**最小可工作的内容后台**，让"写词条 / 调干扰项 / 上传插画 / 扫描课本一键建关卡 / 发布版本 / 一键回滚"全部走 admin REST。客户端从"打包内置 rawfile"升级到"远程拉取 + 本地缓存 + rawfile 兜底"。

### 1.2 一句话设计

> 在 monorepo `server/` 子目录建一个 FastAPI + MongoDB + Vercel 部署的内容后台，纯 REST 暴露 admin 操作 + 公开词包；客户端冷启动远程优先、rawfile 兜底；OpenAI 草稿全部入审；按 7 个 walking-skeleton 子版本递进。

### 1.3 成功标准

- V0.5.1 完成时 admin 能登录、Mongo 中有 1 条 seed word、客户端冷启动从远程拿到词包并落盘缓存。
- V0.5.7 完成时 50+ 词条全在 Mongo、版本化发布生效、LLM 草稿审核可用、教程照片一键导入可用、词条插画 / 发音 MP3 在客户端渲染、操作文档完整、rawfile 仅作首次冷装兜底。
- 客户端现有 38+ UI 套件 0 回归。
- 无网络环境下客户端冷启动仍能进入战斗（rawfile 兜底契约不破）。

## 2. 关键决策记录

| 维度 | 选择 | 理由 |
| --- | --- | --- |
| 后台 UI | 纯 FastAPI + REST，无前端 | 单人运营，curl/Postman 足够；推迟 UI 投资 |
| 服务端语言 | Python 3.12 | 用户指定 |
| Web 框架 | FastAPI | 用户指定 |
| 包管理 | uv | 用户指定 |
| 部署 | Vercel | 用户指定 |
| LLM | OpenAI（gpt-4o-mini 文本 / gpt-4o vision） | 用户选择 |
| 数据存储 | MongoDB Atlas（via Vercel Marketplace）+ Vercel Blob | 用户选择，Beanie ODM 与 Pydantic 协同最自然 |
| 鉴权 | 多用户 + JWT（V0.5 仅 admin，V0.6 复用接家长） | 用户选择 |
| 客户端范围 | 包含；V0.5 末闭环远程拉取 | 用户选择 |
| 拆分策略 | Walking Skeleton（先打通 e2e，再加深） | 基础设施风险前置；任意时刻可上线 |

## 3. 总体架构

```text
┌────────────────────────────────────┐         ┌──────────────────────────────────────┐
│  HarmonyOS Client (entry/)         │         │  FastAPI Backend on Vercel           │
│                                    │         │                                      │
│  WordRepository                    │ HTTPS   │  /api/v1/admin/*    (JWT, content)   │
│   ├─ RemoteWordPackService  ◄──────┼─────────┤  /api/v1/auth/login (issue JWT)      │
│   ├─ LocalCache (preferences /     │         │  /api/v1/packs/latest.json (public)  │
│   │             cacheDir)          │         │  /api/v1/packs/{ver}.json  (public)  │
│   └─ rawfile fallback (cold start) │         │  /api/v1/health                      │
│                                    │         │                                      │
│  Cold-start sequence:              │         │  Layered:                            │
│  1) Try local cache                │         │   routers / services / repos / models│
│  2) If empty: load rawfile         │         │   ↓                                  │
│  3) Async fetch /packs/latest.json │         │  MongoDB Atlas (Beanie ODM, async)   │
│  4) On 200: persist + swap on next │         │   ├─ users         JWT auth          │
│     cold start                     │         │   ├─ words         editable entities │
│  5) On error: keep cache           │         │   ├─ categories    分类元数据        │
│                                    │         │   ├─ word_packs    versioned snapshot│
│                                    │         │   ├─ pack_pointer  current/rollback  │
│                                    │         │   ├─ llm_drafts    单词级草稿状态机  │
│                                    │         │   └─ lesson_import_drafts  课本导入  │
│                                    │         │                                      │
│                                    │         │  Vercel Blob (binary assets)         │
│                                    │         │   ├─ /illustrations/{wordId}-*.png   │
│                                    │         │   ├─ /audio/{wordId}-*.mp3           │
│                                    │         │   └─ /lessons/{ts}-*.jpg             │
│                                    │         │                                      │
│                                    │         │  OpenAI API (server-side only)       │
└────────────────────────────────────┘         └──────────────────────────────────────┘
```

**关键约束：**

- 服务端是 monorepo 子目录 `server/`（已存在），**不分两个仓库**。
- Vercel 上是**一个项目**；FastAPI 通过 vercel python runtime 暴露 ASGI app。
- 客户端从不直接调 OpenAI；LLM 严格只在服务端运行。
- 客户端永远保留 rawfile 作为冷装兜底，**离线可用**契约不破坏。
- 词包发布走两层：DB 中 `word_packs` 集合是源头；公开 endpoint 实时序列化为 JSON 返回（Blob CDN 副本是 V0.6+ 优化路径）。

## 4. 技术栈与仓库布局

### 4.1 服务端依赖（写进 `server/pyproject.toml`，uv 管理）

| 类别 | 包 | 用途 |
| --- | --- | --- |
| Web 框架 | `fastapi` | REST 路由 |
| ASGI | Vercel python runtime 原生支持 | 无需 mangum |
| MongoDB ODM | `beanie` + `motor` | 异步文档模型，类型安全 |
| 鉴权 | `python-jose[cryptography]` + `passlib[bcrypt]` | JWT 签发 + 密码哈希 |
| 配置 | `pydantic-settings` | 从环境变量读密钥 |
| LLM | `openai` (官方 SDK ≥ 1.x) | 文本 + vision |
| Blob 上传 | Vercel Blob HTTP API（`httpx` 直调） | 不需 SDK |
| 测试 | `pytest` + `pytest-asyncio` + `httpx` + `mongomock-motor` | 单元 + e2e |
| Lint / 格式 | `ruff` + `ruff format` | 替代 black / flake8 / isort |
| 类型检查 | `mypy` (strict on) | 静态保证 |

Python 3.12（Vercel python runtime 当前支持 3.9 / 3.12，选 3.12 拿到最新性能 + typing 特性）。

### 4.3 客户端依赖

V0.5.1 起，客户端只新增**一组本地服务类**，不引入新的 oh-package 依赖：

- `services/RemoteWordPackService.ets` — HTTP fetch + 错误归一
- `services/WordPackCache.ets` — preferences-based 缓存器（参考 `WishlistStore` 模式）
- `services/WordPackBootstrapper.ets` — cold-start 三段决策

**不动**：现有 `WordRepository` 的对外接口；它只是被注入新的数据源。

## 5. 子版本拆分（7 个）

```text
V0.5.1  Walking Skeleton          (~5 天)
V0.5.2  词条 CRUD + rawfile 迁移
V0.5.3  版本化词包 + 回滚
V0.5.4  LLM 单词级草稿（干扰项 / 例句）
V0.5.5  教程照片 → 一键分类导入   ★ 新增能力
V0.5.6  词条插画 + 发音 MP3
V0.5.7  收尾（监控 / 文档 / rawfile 退化）
```

### 5.1 V0.5.1 — Walking Skeleton

**目标**：服务端可部署、可登录、可拉到 1 条 seed 词条；客户端在冷启动里走"远程优先 + rawfile 兜底"通路。

| 项 | 内容 |
| --- | --- |
| 服务端 | FastAPI + uv 初始化；`api/index.py` thin entry；`vercel.json` python runtime；Mongo 连接；users / words 两个集合（1 个 seed admin + 1 个 seed word `fruit-apple`）；`POST /auth/login` 返回 JWT；`GET /health`；`GET /packs/latest.json` 直接从 `words` 集合实时打包成 `{ version: 1, words: [...] }` |
| 客户端 | 新建 `RemoteWordPackService.ets`、`WordPackCache.ets`、`WordPackBootstrapper.ets`；`WordRepository.load()` 内部三段：cache → rawfile → 异步刷新；URL 由 BuildProfile 注入 |
| 配置 | Vercel 项目链接 + 环境变量：`MONGO_URI` `JWT_SECRET` `ADMIN_BOOTSTRAP_USER` `ADMIN_BOOTSTRAP_PASS` |
| 验收 | (1) `vercel deploy` 成功；(2) `curl POST /auth/login` 拿到 JWT；(3) `curl /packs/latest.json` 返回包含 `fruit-apple` 的合法 JSON；(4) HarmonyOS 客户端冷启动后 preferences 中可见缓存；(5) 飞行模式冷启动可走 rawfile 兜底不崩 |

**明确不做**：词条 CRUD 写接口、版本号语义、回滚、LLM、Blob、注册流程。

### 5.2 V0.5.2 — 词条 CRUD + rawfile 迁移

**目标**：50 条 rawfile 词全部进 Mongo；admin 可通过 REST 增删改；客户端无需改动即可吃到新词。

| 项 | 内容 |
| --- | --- |
| 服务端 | `POST/GET/PUT/DELETE /admin/words` 全套（JWT 必需）；`scripts/seed_from_rawfile.py` 解析 `entry/src/main/resources/rawfile/data/words_v1.json` 做幂等 upsert（按 `word.id`） |
| 数据 | `words` 集合添加索引：`id`（唯一）、`category`、`difficulty` |
| 测试 | `tests/test_words.py` 覆盖增删改 + 鉴权失败 401 + 重复 id 409 |
| 客户端 | **零改动**；只需重新冷启动验证拉到的词数从 1 → 50 |
| 验收 | (1) seed 后 Mongo 中 50 条词；(2) admin 能 POST 第 51 条词，`/packs/latest.json` 立即返回 51 条；(3) 客户端冷启动后战斗页词池扩到 51 条；(4) 删词后客户端再冷启动看到 50 条 |

**明确不做**：版本号、回滚、草稿态。每次 admin 改动都是"立即生效"。

### 5.3 V0.5.3 — 版本化词包发布 + 回滚

**目标**：从"实时打包"升级到"快照发布"模型；admin 显式 publish 才让线上变；可一键回滚。

| 项 | 内容 |
| --- | --- |
| 服务端 | 新集合 `word_packs`（`{ version, words[], schema_version, published_at, published_by, notes? }`）；指针文档 `pack_pointer`（`{ singleton_key="main", current_version, previous_version }`）；`POST /admin/packs/publish` 快照 + 版本 +1 + 更新指针；`POST /admin/packs/rollback` 把指针指回 `previous_version`；`GET /admin/packs` 列所有版本；`/packs/latest.json` 改为读指针对应快照；`/packs/{version}.json` 任意版本直拉；响应附 `ETag` header（pack id）支持 304 |
| 客户端 | `WordPackCache` 新增 `cachedVersion` + `cachedEtag` 字段；冷启动时用 `If-None-Match` 条件请求；304 不替换缓存；新版本时持久化整包再生效 |
| 验收 | (1) V0.5.2 状态发版得 v1；(2) 改一条词再发版得 v2，`/packs/latest.json` 是 v2；(3) rollback 后回到 v1；(4) 客户端 304 命中时无下载流量 |

**明确不做**：发版定时调度、CDN 预热、变更审计 log。

### 5.4 V0.5.4 — LLM 单词级草稿（干扰项 + 例句）

**目标**：admin 一键让 OpenAI 给某词生成干扰项 / 例句，进入 pending 待审；approve 才合入词条；reject 丢弃。

| 项 | 内容 |
| --- | --- |
| 服务端 | 新集合 `llm_drafts`（详见 §6.5）；`POST /admin/words/{id}/generate-distractors` 调 gpt-4o-mini，强制 JSON Schema 输出，写 draft；`POST /admin/words/{id}/generate-example` 同；审核流 `GET /admin/drafts`、`POST .../approve`、`POST .../reject`、`PATCH .../{id}` |
| 词条扩字段 | `distractors: list[str]?`、`example_sentence_en: str?`、`example_sentence_zh: str?` |
| 词包 | snapshot schema bump 到 v2（向后兼容，老客户端忽略） |
| 客户端 | `WordEntry` 加可选字段（仅类型 + parser 兼容，**当前不使用**） |
| 验收 | (1) 给 apple 生成干扰项得 ≥3 词且非 "apple"；(2) approve 后 word 实体含字段；(3) 重发版后快照 v3 含字段；(4) reject 后 word 不变；(5) OpenAI 失败时 503 + 友好错误 |

**明确不做**：批量生成（V0.6+）、剧情文案生成（V0.7）、审核操作日志。

### 5.5 V0.5.5 — 教程照片 → 一键分类导入

**目标**：admin 上传一张课本/教程页照片 → OpenAI Vision（gpt-4o）解析词表 + 生成分类标题 + 故事段 → 进入待审；approve 后批量创建 category + words。

| 项 | 内容 |
| --- | --- |
| 服务端 - 上传 | `POST /admin/lessons/import` (multipart) → 原图先丢 Vercel Blob 留审计（路径 `lessons/{ts}-{rand8}.jpg`）→ 拿到 Blob URL 后调 OpenAI vision，**强制 JSON Schema 结构化输出** → 落 `lesson_import_drafts` 集合 |
| LLM 输出契约 | `{ "category_id": "school-supplies", "label_en": "School Supplies", "label_zh": "学校用品", "story_zh": "...80-150字童话风简介...", "words": [{ "word": "pencil", "meaningZh": "铅笔", "difficulty": 1 }, ...] }` |
| 服务端 - 审核 | `GET /admin/lesson-drafts?status=pending`；`POST .../approve` 触发 batch upsert（创建 Category 实体 + 批量 upsert Words，同名 word 跳过并在 response 列出冲突）；`POST .../reject` 丢弃；`PATCH .../edit` 允许 admin 在 approve 前微调 JSON |
| 新实体 | `Category`：`{ id, label_en, label_zh, story_zh?, source_image_url?, source: "manual" \| "lesson-import", created_at, updated_at }`；服务端 startup hook 跑 idempotent 回填，把现有 5 个硬编码分类（fruit / place / home / animal / ocean）写入为 manual 来源，预填 label_en / label_zh，story_zh 留空（admin 后续手填或借 LLM 单条生成工具，V0.6+） |
| 词包 JSON | bump 到 schema_v4，顶层加 `categories: [...]`（参见 §6.7） |
| 客户端 | 读取 `categories[]`；在 HomePage region 卡片下方展示 `storyZh`（小字 ≤2 行）；新分类**不会自动出现在 region picker** —— `AdventureCatalog` 仍是硬编码 region 列表，新建 region 是 V0.6+ 工作；V0.5.5 客户端只渲染"已存在 region 的故事文案" |
| 验收 | (1) 上传 1 张含 8-15 个英文单词的课本页 → draft 中词数 ≥80% 准确率；(2) `PATCH .../edit` 改完一个 typo 后再 approve 不重复创建词；(3) approve 后 category 实体 + words 实体存在；(4) 重新发版后 `/packs/latest.json` schema_v4 含新 category + 新 words；(5) 客户端冷启动后已存在 region（如 fruit-forest）卡片下方出现 `story_zh` 文案；(6) 同 word 二次导入返回冲突清单不写库 |

**Vercel timeout 约束**：vision 调用在 Vercel Function 上有 60s timeout（hobby 10s / pro 60s）。一张含 15 个词的课本页 gpt-4o vision 通常 5-15s。**V0.5.5 设计上就是同步**：`POST /admin/lessons/import` 阻塞等待 vision 完成，立刻返回 `extracted` 完整结果。如果落地时发现 hobby 计划超时，改成"先存 Blob + draft 占位 + 后台任务（Vercel Cron / QStash）"两步走 —— 留实施期决定。

**明确不做**：自动创建 region / `AdventureCatalog` 数据化（V0.6+）；多语种识别（仅英→中）；批量上传多张图（一次一张）；OCR-only 路径；课本来源版权追踪。

### 5.6 V0.5.6 — 词条插画 + 发音 MP3

**目标**：admin 上传词条插画 PNG 和发音 MP3；客户端下载缓存并渲染/播放。

**明确不做**：Blob 全量预下载（lazy on demand）、图片自动压缩、音频转码、CDN 防盗链。

### 6.1 `users`（V0.5.1 起）

```python
class User(Document):
    username: Indexed(str, unique=True)
    password_hash: str                # bcrypt
    role: Literal["admin", "parent"]  # V0.5 只用 admin；parent 给 V0.6 留
```

### 6.2 `words`（V0.5.1 → V0.5.6 渐进扩展）

```python
class Word(Document):
    id: Indexed(str, unique=True)        # 业务 id, 对齐客户端 "fruit-apple"
    word: str
    meaningZh: str
    category: Indexed(str)               # fruit / place / home / animal / ocean / ...
    difficulty: int                      # 1-5

    # V0.5.4 起追加（可选）
    distractors: list[str] | None = None
    example_sentence_en: str | None = None
    example_sentence_zh: str | None = None

    # V0.5.6 起追加（可选）
    illustration_url: str | None = None
    audio_url: str | None = None

    created_at: datetime
    updated_at: datetime

    class Settings:
        name = "words"
        indexes = ["category", "difficulty", [("category", 1), ("difficulty", 1)]]
```

### 6.3 `categories`（V0.5.5 起）

```python
class Category(Document):
    id: Indexed(str, unique=True)        # 业务 id, kebab-case ("fruit", "school-supplies")
    label_en: str
    label_zh: str
    story_zh: str | None                 # 80-150 字童话风简介，可空
    source_image_url: str | None         # 仅 lesson-import 来源有值
    source: Literal["manual", "lesson-import"]
    created_at: datetime
    updated_at: datetime

    class Settings:
        name = "categories"
```

V0.5.5 startup hook 中跑 idempotent upsert（按 `id`），回填 5 条 manual：

```text
fruit         → "Fruit"           "水果"
place         → "Places"          "地点"
home          → "Home"            "家"
animal        → "Animals"         "动物"
ocean         → "Ocean"           "海洋"
```

`story_zh` 落地时空着；admin 后续手填或借 LLM 单条生成（留 V0.6+ 工具）。

### 6.4 `word_packs` + `pack_pointer`（V0.5.3 起）

```python
class WordPack(Document):
    version: Indexed(int, unique=True)   # 单调递增
    words: list[dict]                    # 发版瞬间从 Word 全表 dump 的快照
    categories: list[dict] | None        # V0.5.5 起填，老快照为 None
    schema_version: int                  # 1 / 2 / 4 / 5（v3 跳过，见 §6.7）
    published_at: datetime
    published_by: str
    notes: str | None

    class Settings:
        name = "word_packs"


class PackPointer(Document):
    singleton_key: Literal["main"] = "main"
    current_version: int
    previous_version: int | None

    class Settings:
        name = "pack_pointer"
        indexes = [{"key": [("singleton_key", 1)], "unique": True}]
```

存"完整冗余快照"而不是"diff"，因为：

- 客户端拉取需要单文件即可生效。
- Mongo 文档大小上限 16MB，50 → 5000 词都装得下。
- 回滚 = 切指针，**不是**回放 diff，零路径分歧。

### 6.5 `llm_drafts`（V0.5.4 起）

```python
class LlmDraft(Document):
    target_word_id: Indexed(str)         # 对应 Word.id
    draft_type: Literal["distractors", "example"]
    content: dict                        # 由 draft_type 决定 shape
    status: Literal["pending", "approved", "rejected"]
    created_at: datetime
    reviewed_at: datetime | None
    reviewer: str | None
    model: str                           # 例 "gpt-4o-mini-2024-07-18"
    prompt_version: int

    class Settings:
        name = "llm_drafts"
        indexes = [[("status", 1), ("created_at", -1)]]
```

`content` 形状（应用层断言）：

```text
draft_type=distractors:  { "distractors": ["banana","cherry","grape"] }
draft_type=example:      { "en": "I eat an apple every day.", "zh": "我每天吃一个苹果。" }
```

### 6.6 `lesson_import_drafts`（V0.5.5 起）

```python
class LessonImportDraft(Document):
```

**前向兼容契约：**

- 客户端 parser 只读已知字段，未知字段忽略 — 老客户端拉到 v5 包不崩。
- 服务端必须**单调递增 schema_version**；不在已发布版本回头改字段语义。
- 客户端按 `schema_version` gate 新功能（例如插画渲染只在 `schema_version >= 5` 时启用）。
- v3 跳号是有意为之 —— 把"加 categories 顶层结构"和"加每词 URL 字段"分到不同版本号；跳号不影响兼容。

## 8. 客户端集成

### 8.1 模块边界

```text
WordRepository (existing)
    ↓ load()
WordPackBootstrapper          ← 新增 V0.5.1，编排冷启动 3 段决策
    ├─ WordPackCache          ← 新增 V0.5.1，preferences 持久化
    ├─ rawfile reader (existing, fallback only)
    └─ RemoteWordPackService  ← 新增 V0.5.1，HTTP fetch + parse + 错误归一
```

### 8.2 冷启动序列（伪代码）

```text
function bootstrapWords():
    # 同步段，必须返回可用 WordRepository
    cache = WordPackCache.read()
    if cache.exists() and cache.schemaCompatible():
        repo = WordRepository.fromPackJson(cache.json)
    else:
        repo = WordRepository.fromRawfile()

    # 异步段，不阻塞首屏
    fireAndForget(async () => {
        try:
            etag = cache.etag if cache.exists() else null
            resp = RemoteWordPackService.fetchLatest(ifNoneMatch=etag, timeout=8s)
            if resp.status == 304:
                return
            if resp.status == 200:
                if !validatePackJson(resp.body): return
                WordPackCache.write(resp.body, etag=resp.etag)
                # 不立即热替换 repo —— 留待下次冷启动生效，避免战斗中切词池
        catch any:
            log only, never crash
    })

    return repo
```

**关键不变量：**

- 首屏永不被网络阻塞（rawfile / cache 永远在 0ms 内可用）。
- 战斗会话进行中的 `WordRepository` 实例**不被替换**。
- 网络结果只影响"下次冷启动看到什么"，绝不影响"本次冷启动看到什么"。

### 8.3 各子版本客户端改动

### 8.4 schema_version gate（V0.5.5+）

```text
if pack.schema_version >= 4:
    enable category story rendering

    silent fallback
```

## 9. Vercel 部署拓扑

### 9.1 单 Vercel 项目，Python runtime

```text
happyword (Vercel project)
├── Production (linked to git main branch)
├── Preview (auto for every PR)
└── Development (`vercel dev` 本地)
```

`server/vercel.json`（最小可工作版）：

```json
{
  "version": 2,
  "builds": [
    { "src": "api/index.py", "use": "@vercel/python", "config": { "maxLambdaSize": "50mb" } }
  ],
  "routes": [
    { "src": "/api/(.*)", "dest": "/api/index.py" },
    { "src": "/(.*)", "dest": "/api/index.py" }
  ],
  "functions": {
    "api/index.py": { "memory": 1024, "maxDuration": 60 }
  }
}
```

`server/api/index.py`：

```python
from app.main import app  # FastAPI instance — Vercel python runtime auto-detects ASGI
```

### 9.2 vercel.json 位置

`server/` 是子目录，但 Vercel 项目根需要在那里。**推荐方式**：在 Vercel 项目设置里把 **Root Directory** 设为 `server`。`vercel.json` 也放 `server/vercel.json`。git push 触发部署只在 `server/**` 改变时跑（用 Vercel "Ignored Build Step" 的 git diff 命令做）。

### 9.3 环境变量

| 变量 | 用途 | 示例 / 来源 |
| --- | --- | --- |
| `MONGO_URI` | Mongo Atlas 连接 | Vercel Marketplace MongoDB integration 自动注入 |
| `MONGO_DB_NAME` | 数据库名 | `happyword` / `happyword_dev` |
| `JWT_SECRET` | JWT HS256 签名密钥 | `openssl rand -hex 32` |
| `JWT_EXPIRE_HOURS` | token 有效期 | `24` |
| `ADMIN_BOOTSTRAP_USER` | 启动 hook 创建首个 admin 的用户名 | `admin` |
| `ADMIN_BOOTSTRAP_PASS` | 启动 hook 创建首个 admin 的密码 | 强密码 |
| `OPENAI_API_KEY` | OpenAI 密钥 | `sk-...` |
| `OPENAI_MODEL_TEXT` | 文本任务模型 | `gpt-4o-mini` |
| `OPENAI_MODEL_VISION` | vision 任务模型 | `gpt-4o` |
| `BLOB_READ_WRITE_TOKEN` | Vercel Blob token | Vercel Storage 创建后注入 |
| `CORS_ALLOW_ORIGINS` | 允许的客户端来源 | V0.5.1 设 `*`，V0.6+ 收紧 |
| `LOG_LEVEL` | 日志等级 | `info` / `debug` |

Production / Preview / Development **三套独立**的 `MONGO_DB_NAME` —— 不共用一个数据库；preview 部署不污染生产数据。

### 9.4 本地开发流



### 9.6 客户端 ↔ 服务端 BaseURL 对照

| 环境 | `SERVER_BASE_URL` | 用途 |
| --- | --- | --- |
| 客户端 dev / 模拟器 | `http://10.0.2.2:8000` | 连本地 uvicorn |
| 客户端 dev / 真机 | `https://<你的隧道>.trycloudflare.com` | 临时联调 |
| 客户端 staging | `https://happyword-preview-xxx.vercel.app` | preview deploy |
| 客户端 prod | `https://happyword.vercel.app` 或自定义域 | 生产 |

V0.5.1 先写死 prod；dev 切换通过 BuildProfile 不同 build 类型完成。

## 10. 风险登记

| # | 风险 | 影响 | 缓解 |
| --- | --- | --- | --- |
| R1 | Vercel Function 60s 超时撞上 vision 慢响应（pro 计划 60s，hobby 10s） | V0.5.5 lesson import 长图片 / 慢网会失败 | 同步设计 + LLM 客户端 8s timeout + 失败 draft 入 `failed` 状态；hobby 计划留逃生口（异步队列）但不在 V0.5 范围 |
| R2 | Mongo Atlas 免费层冷启动延迟（M0 共享集群 ≥1s） | 首请求慢，影响客户端冷启动体感 | 客户端 fetch 异步、不阻塞首屏；服务端 startup hook 预热连接；考虑 V0.5.7 升级到 M2 付费层 |
| R3 | 客户端缓存被异常包污染导致冷启动崩溃 | 用户必须卸载重装 | `WordPackCache.read()` 严格 schema 校验；解析失败立即 wipe + 走 rawfile；失败计数 ≥3 自动切回 rawfile 并丢弃缓存 |
| R4 | OpenAI API 故障 / 限流 / 成本失控 | LLM 草稿生成全停 | 服务端缓存 prompt → output（按 word.id + prompt_version + model 做 dedupe）；单日调用上限环境变量；指标记录调用次数和总 token |
| R5 | Vercel 部署链路（git push → build）失败 | 服务端无法上线 | Preview deploy 验证；保留 `vercel rollback` 一键回退上一个生产版本 |
| R6 | JWT secret 泄漏 | 任何人都能签发管理 token | secret 仅 Vercel env vars；commit 钩子拒绝 `.env.local` 入 git；定期轮换（≥半年） |
| R7 | rawfile schema 与远程 schema 漂移 | 客户端 fallback 时字段缺失 | 服务端 V0.5.2 迁移之后即视 rawfile 为 read-only baseline；不再修改 rawfile，所有改动走 admin API |
| R8 | Lesson import vision 误识别（OCR 错字、多语种混入） | admin 修订成本高 | `PATCH /lesson-drafts/{id}` 支持完整 JSON 编辑；prompt 强约束输出 schema + 拒绝非英文词 |
| R9 | Mongo Atlas 数据丢失（误删 / 故障） | 词包数据全丢 | V0.5.7 加 `scripts/backup_pack.py` 定期备份 word_packs 集合；Atlas 自带 7 天 PITR 但免费层无；上线前确认存档机制 |
| R10 | 客户端老版本 + 新 schema 不兼容 | 老用户无法升级体验 | schema_version gate 严守"未知字段忽略 + 已知字段强约束"；测试矩阵覆盖 client 旧 × server 新两条路径；客户端发版 changelog 标注最低 schema 兼容 |
| R11 | Blob 资源外泄 | 内容资产被扒 | V0.5 暂接受公开访问（CDN 友好）；V0.6+ 视情况加签名 URL |
| R12 | Vision 调用费用（gpt-4o 每张图 $0.01-0.05） | 长期成本 | admin 操作低频（人工触发）；按月统计；超阈值告警留 V0.5.7 stats endpoint |
