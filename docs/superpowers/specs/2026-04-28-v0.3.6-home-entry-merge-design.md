## 1. 范围与架构概览

### 1.1 V0.3.6 一句话

> 砍掉首页右侧 "开始游戏 / 复习错题" 双按钮竖列；今日冒险卡片的红色 CTA 升级为唯一主入口；复习入口以一个 📚 圆形按钮迁到顶部工具栏，未解锁时点击出 inline toast；顺便清掉只为开发期诊断而生的 🧪 TTS 测试入口与 `TtsTestPage`。

### 1.2 单 spec 单 plan

V0.3.6 是产品收敛 + 死代码裁剪，仅触一页 + 一个独立测试页 + 一个路由清单：

```text
1. 删除 TtsTestPage 入口（最小风险，先做）
2. 合并主入口：删 HomePage 右列；将今日冒险 CTA 重命名为 HomeStartButton；删 enterNormalMode
3. 复习入口迁顶栏：HomeReviewButton 移到顶部工具栏（📚），新增锁态 inline toast + UI 测试
```

按 "先小后大、先无测试影响后改测试" 顺序落地，**3 个独立 commit**，可分别 revert。

### 1.3 模块归属

```text

pages/TtsTestPage.ets            整文件删除
resources/.../main_pages.json    移除 "pages/TtsTestPage" 路由

```

### 1.4 关键不变量（必须保持）

- `models/GameConfig.ets` 三个模式常量 (`GAME_MODE_NORMAL`/`REVIEW`/`TODAY`) 不动；常量本身不删。
- `BattlePage.ets` 模式分发逻辑（`isReviewMode` / `isTodayMode` 派生 + 战斗结束重置回 `NORMAL`）不动。
- `enterReviewMode()` / `enterTodayAdventure()` 方法签名与行为不变；唯独删除 `enterNormalMode()`（孤儿）。
- `HomeReviewButton` 的 stable id **复用**，其 onClick 行为对解锁态保持不变（仍走 `enterReviewMode`）。
- 已有 6 个 UI 套件（RoutingFlow / MonsterRoster / MagicAttack / SpeakerButton / CritSpectacle / ReviewMode）通过 `HomeStartButton` 进入战斗；该 id 仍存在但绑定从 normal-mode 改为 today-mode。BattlePage 渲染同样的 BattleTitle/Prompt/SpeakerButton 等 id，套件主体断言不变。
- 适用 codelinter 规则 `avoid-overusing-custom-component-check`：HomePage 沿用 @Builder 内联，不拆 @Component。

### 2.4 Locked review toast

- 触发：用户点击 `HomeReviewButton` 时若 `recentWrongCount < REVIEW_MIN_WRONG_IDS`(3)。
- 渲染：在外层 `Stack` 上新增 `Alignment.Top` 子节点（与右上角顶栏并列）；条件渲染 `if (this.reviewLockedToastVisible)`。
- 节点：单 `Text` 包了一层 padding/border：
  - 文案：`先答错几题再来复习吧`
  - id: `HomeReviewLockedToast`
  - 样式：`fontSize 14`、`fontColor '#FFFFFF'`、`backgroundColor '#3A3A3ACC'`、`borderRadius 18`、`padding({ left: 16, right: 16, top: 8, bottom: 8 })`、`margin({ top: 96 })`
- 时序：
  - 点击进入解锁判定，未解锁 → 调 `showReviewLockedToast()`：
    - 若 `private toastTimer >= 0`，先 `clearTimeout(this.toastTimer)`
    - `this.reviewLockedToastVisible = true`
    - `this.toastTimer = setTimeout(() => { this.reviewLockedToastVisible = false; this.toastTimer = -1; }, 1500)`
  - `aboutToDisappear()` 中守卫地清 timer，避免组件卸载后回调写 state。
- 不阻塞点击：toast 是顶部薄条，覆盖区在标题上方；下层 CTA 仍可点。

### 2.5 入口路由汇总

| 用户动作 | HomePage 方法 | cfg.mode 写入 | 后续页面 |
|---|---|---|---|
| 点 `HomeStartButton` | `enterTodayAdventure()` | `GAME_MODE_TODAY` | BattlePage（今日 5 关）|
| 点 `HomeReviewButton`（解锁） | `enterReviewMode()` | `GAME_MODE_REVIEW` | BattlePage（错题复习）|
| 点 `HomeReviewButton`（锁定） | `showReviewLockedToast()` | 不写 | 留在 HomePage |
| 点 `HomeWishlistButton` | router.pushUrl WishlistPage | 不写 | WishlistPage |
| 点 `HomeConfigButton` | router.pushUrl ConfigPage | 不写 | ConfigPage |

约定（与现行套件一致）：

- `it(name, 0, async (done: Function) => { ... done(); })` 签名

2 个用例：

```typescript
import { describe, it, expect } from '@ohos/hypium';
import { abilityDelegatorRegistry, Driver, ON, Component } from '@kit.TestKit';
import { Want } from '@kit.AbilityKit';
import { clickByIdShared } from './RoutingFlow.ui.test';

const BUNDLE: string = 'com.terryma.wordmagicgame';
const DELEGATOR: abilityDelegatorRegistry.AbilityDelegator =
  abilityDelegatorRegistry.getAbilityDelegator();

async function launchApp(): Promise<Driver> {
  const want: Want = { bundleName: BUNDLE, abilityName: 'EntryAbility' };
  await DELEGATOR.startAbility(want);
  const driver: Driver = Driver.create();
  await driver.delayMs(1000);
  return driver;
}

export default function homeToolbarLockedUiTest(): void {
  describe('HomeToolbarLocked', () => {
    it('topToolbarHasReviewNotTts', 0, async (done: Function) => {
      const driver: Driver = await launchApp();
      await driver.assertComponentExist(ON.id('HomeReviewButton'));
      const tts: Component | null =
        await driver.findComponent(ON.id('HomeTtsTestButton'));
      expect(tts === null).assertTrue();
      done();
    });

    it('lockedReviewShowsToastAndStays', 0, async (done: Function) => {
      const driver: Driver = await launchApp();
      // Fresh cold launch → 0 wrong words → review locked.
      await clickByIdShared(driver, 'HomeReviewButton');
      await driver.delayMs(300);
      await driver.assertComponentExist(ON.id('HomeReviewLockedToast'));
      await driver.assertComponentExist(ON.id('HomeStartButton'));
      await driver.delayMs(1800);
      const toast: Component | null =
        await driver.findComponent(ON.id('HomeReviewLockedToast'));
      expect(toast === null).assertTrue();
      done();
    });
  });
}
```

## 6. 风险与回滚

### 6.1 主要风险与对冲

| 风险 | 触发 | 对冲 |
|---|---|---|
| `HomeStartButton` 行为切换导致老 UI 套件挂掉 | 任一隐含 normal-mode 假设 | plan 内 "triage UI test fallout" 任务；fail 一个修一个 |
